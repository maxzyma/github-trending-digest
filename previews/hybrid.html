VectifyAI / PageIndex

无向量、基于推理的 RAG 系统。把文档转成层级树，让 LLM 在树上做推理搜索。

解决什么问题。传统向量 RAG 在专业长文档上表现不佳——向量相似度≠相关性。比如财务报告中"净利润"和"纯利润"向量很近但语义不同；跨段落推理（"Q1 收入增长但利润率下降的原因"）向量检索无法处理。PageIndex 用树结构 + 推理来解决。

谁在用。金融文档分析（HN 真实用户反馈"significant improvements in retrieval accuracy"）；长文档问答（财报、监管文件、教科书 Colab 演示）；MCP 集成（独立 pageindex-mcp 仓库）。

+ 正面信号
  • HN Show HN 192 分 / 128 评论，今日榜单 HN 热度最高
  • 真实用户反馈 retrieval accuracy 改善
  • fork:star 8.4%，健康度极高
  • Discord + 文档站 + 在线 Chat 生态完整
− 风险信号
  • 推理式检索成本可能比向量贵 10–100×
  • 树搜索串行，无法并行，延迟更高
  • HN 评论质疑"本质还是检索，只是 LLM 路由"
  • 面对 Pinecone/Weaviate 等成熟厂商需证明优势

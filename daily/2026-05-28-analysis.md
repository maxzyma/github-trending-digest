### [p-e-w/heretic](https://github.com/p-e-w/heretic) — NEW ⭐ 22,044（+211）| Python | 信源：强

**它是什么**：一个"全自动给开源大模型去审查"的命令行工具。开源模型（Llama、Qwen、Gemma 等）出厂时被训练成会拒绝某些请求（"安全对齐"），heretic 用一种叫 abliteration 的技术，自动找出并移除模型内部负责"拒绝"的方向，让模型不再拒答，同时尽量不损伤其它能力。作者 p-e-w 是 LLM 工具圈的活跃开发者。

**解决什么问题**：以前给模型去审查要手工分析模型激活、反复试参数，门槛高且容易把模型"弄笨"。heretic 把这个过程全自动化（一行 `pip install heretic-llm`），并在质量保留上做到优于手工——这是本地 LLM 开发者长期的痛点（安全对齐经常误拒合法请求）。

**谁在用、怎么用**（来自 HN / HF / Issues）:
- 场景 1：本地 LLM 爱好者去除模型对合法请求的误拒，构建无审查的本地助手（r/LocalLLaMA 核心用户群）
- 场景 2：研究者用它研究模型行为 / 对齐机制；HuggingFace 上已有 3,500+ 个 Heretic 衍生模型
- 场景 3：`pip install heretic-llm` 一行命令；Llama-3.1-8B / Qwen3-4B 在 RTX 3090 约 45 分钟跑完

**社区怎么评价**:
- 👍 正面：HN 主帖 745 分 / 380 评，曝光极高；独立基准（MMLU/GSM8K）验证质量保留占优；被推荐为"积极维护、外科手术式、低 KL"的成熟方案（对照竞品 OBLITERATUS）
- 👎 负面：部分用户反映 abliterated 模型仍偶现"变笨"；少数 HN 评论批 README 有 AI 生成痕迹
- 📊 健康度：22,044★ / 89 subscribers（247:1）/ 2.4K forks / 74 open issues / 22 contributors（核心 p-e-w 102 commits + anrp/accemlcc/Vinayyyy7 等多名外部贡献者实现 LoRA 引擎 / 多轮试验）/ 2026-05-22 仍有推送 / AGPL-3.0

**风险与争议**:
- **safety/misuse**: FT 调查——已生成 3,500+ 去审查模型、下载 1,300 万次，Gemma 4 发布 90 分钟内被破解；Futurism 报道可诱导输出氯气攻击等危害指令；作者明确承认"工具中立、用途自负"

- **未证实信息**: Reddit 一帖称"Heretic 被 Meta 发律师函"（43 分但仅 1 评论），搜索无佐证，**疑似标题党/讽刺，未证实**
- **vanity（伪）**: 247:1 看似空心，但 HN 745 分 + 媒体调查 + HF 3500+ 衍生模型多源交叉证明真实影响力，是高 ratio 但非 vanity 的反例

**技术备注**：Python / abliteration（移除 refusal direction）+ LoRA 引擎 / 支持多数 dense transformer + 部分 MoE / AGPL-3.0 / `pip install heretic-llm`。

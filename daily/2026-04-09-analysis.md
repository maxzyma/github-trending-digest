### [newton-physics/newton](https://github.com/newton-physics/newton) — NEW ⭐ 4,103（+91）| Python (NVIDIA Warp) | 信源：中

**它是什么**: 一个 GPU 加速的物理仿真引擎——让机器人研究者能在显卡上高速模拟物理世界（关节运动、碰撞、柔性体等），速度比传统 CPU 仿真快 70~100 倍。由 Disney Research、Google DeepMind 和 NVIDIA 三方发起，现为 Linux Foundation 项目。

**解决什么问题**: 训练机器人需要海量物理仿真数据，传统工具（MuJoCo、PyBullet）在 CPU 上跑太慢。NVIDIA 自家的 IsaacSim/IsaacLab 又过于庞大复杂。Newton 定位在两者之间——轻量但 GPU 原生，继承了 Warp 的 `warp.sim`（已废弃）并整合 MuJoCo Warp 作为后端。

**谁在用、怎么用**:
- Skild AI：GPU 机架装配（连接器插入、板件放置）
- Samsung + Lightwheel：冰箱产线柔性电缆操控
- Toyota Research Institute：求解器开发与接触建模
- HN 用户评价"IsaacSim 的更好替代，更轻量可定制"（HN 130 points / 32 comments）

**社区怎么评价**:
- 👍 HN 社区认可其定位——填补 IsaacSim（太重）和 PyBullet（太老）之间的空白
- 👍 Apache-2.0 + Linux Foundation 治理，开源治理透明
- 👎 HN 批评"本质是 mujoco-warp 的薄封装，创新度存疑"
- 👎 仅 Python API，缺 C/C++ 接口引发机器人社区不满
- 👎 命名与已有的 Newton Dynamics 引擎冲突
- 📊 Stars/Watchers 比 100:1（偏高）；192 个 open issues；top 贡献者均为 NVIDIA/Google 员工，社区自治度低；X 上仅官方推广，Reddit 无讨论

**风险与争议**: NVIDIA GPU 锁定——非 NVIDIA 硬件无法使用（macOS 仅 CPU 模式）。项目不到一年，issue 积压速度快于消化速度。"大厂发起 → 社区接盘"模式能否持续取决于 NVIDIA 的长期投入意愿。

**技术备注**: Python + NVIDIA Warp + MuJoCo Warp 后端；支持 OpenUSD、可微分仿真；Apache-2.0

### [goharbor/harbor](https://github.com/goharbor/harbor) — NEW ⭐ 28,128（+42）| Go | 信源：中

**它是什么**: 企业级容器镜像仓库——存储、签名、扫描 Docker/OCI 镜像的一站式平台。相当于"企业私有的 Docker Hub"，带 RBAC 权限控制、漏洞扫描、镜像跨区域复制等功能。CNCF 毕业项目，10 年历史。

**解决什么问题**: 企业不想把容器镜像放在公有 Docker Hub 上（安全合规要求），也不想依赖云厂商锁定（ECR/ACR/GCR）。Harbor 提供自托管方案，尤其适合金融、政府、air-gapped 隔离网络等场景。

**谁在用、怎么用**:
- 金融行业（13% 用户占比）、政府机构的私有镜像仓库
- Kubernetes 平台团队标配组件，配合漏洞扫描 + 镜像签名
- 多数据中心跨区域镜像复制
- 新方向：Harbor Satellite 扩展至边缘/IoT 场景

**社区怎么评价**:
- 👍 CNCF 毕业项目，治理成熟，10 年历史验证
- 👍 #22311 全面 Multi-Arch 支持需求（34 评论）——说明生产用户群体庞大
- 👎 #4982 SAML 认证需求 **8 年未关闭**（71 评论）——企业刚需长期被忽视
- 👎 #21062 删除 tag 时 PostgreSQL 死锁——生产环境影响大
- 👎 #14774 BuildKit cache export 404（64 评论）——CI/CD 集成痛点
- 📊 827 open issues；HN/Reddit/X 几乎无讨论（信源集中在 GitHub Issues）

**风险与争议**: 市场份额从 28.3% 降至 26.1%，云厂商托管 registry 持续蚕食自托管市场。运维负担重（需维护 PostgreSQL + Redis + 存储后端）。harbor-operator 已官方标记 DEPRECATED。今日 +42 stars 属于异常低增量上榜，可能是 Trending 算法波动。

**技术备注**: Go；Apache-2.0；CNCF Graduated

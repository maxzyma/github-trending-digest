### [zilliztech/claude-context](https://github.com/zilliztech/claude-context) — NEW ⭐ 6,653（+169）| TypeScript | 信源：中

**它是什么**: 一个给 Claude Code、Cursor 等 AI 编程助手的"代码搜索插件"（MCP Server）——把你整个代码库先建成向量索引存到 Zilliz/Milvus 向量库里，Claude 问问题时只取相关片段送进上下文，不用每次把整个目录塞进去。官方定位是"把百万行代码都当 Claude 的上下文用"。Zilliz 自家出品，顺带给自家向量云带流量。

**解决什么问题**: 大型代码库让 AI 编程助手要么"看不全"（窗口不够）要么"看了很贵"（每次塞全目录 token 爆炸）。claude-context 把代码变成向量库里的可检索片段，按需取用——理论上是"用向量检索换 token 成本"。

**谁在用、怎么用**:
- 想在大型 monorepo 里用 Claude Code/Cursor 的开发者（VS Code 插件 + MCP 双形态）
- 但真正跑通的案例稀少——最热 Issue #145（33 评论）核心问题："索引成功了，但 `search_code` 说没索引"，功能断裂 8 个月未修
- #226（12 评论）同类问题再次出现；#101、#81 要求脱钩 OpenAI Embedding，作者未响应

**社区怎么评价**:
- 👍 570 forks 说明愿意试的基数大；VS Code 插件 + npm 双渠道分发完备
- 👎 核心功能 bug 长期 open（#145 挂 8 个月）；11 contributors 中 9 人仅 1-2 次提交，实际只靠 `zc277584121` 一人（108 提交），bus factor=1
- 👎 **战略挤压**: 2025-11 Anthropic 官方推出 MCP Tool Search（lazy loading + 46.9% token 节省）后，Simon Willison 等核心开发者转向官方方案，第三方 code search MCP 生态位被压缩
- 📊 6,653 stars / 35 watchers（190:1），典型"收藏夹项目"——社交证明远超真实追随
- 🔕 HN/Reddit 都无独立讨论，社区传播主要靠 Zilliz 官方渠道

**风险与争议**: 强耦合 OpenAI Embedding + Zilliz/Milvus 向量库（双厂商锁定），多个 issue 反弹但未解耦。与 Anthropic 官方 MCP Tool Search 正面相撞——当原厂开始原生解决 context bloat，第三方方案差异化收窄。社区 fork 替代品激增（claude-context-local、claude-context-mode 等），本身就是痛点证据。

**技术备注**: TypeScript；Node.js 20+（< 24）；MIT；依赖 Zilliz Cloud（或自建 Milvus）+ OpenAI Embedding；提供 MCP Server + VS Code 扩展双形态

**谁在用、怎么用**:

**社区怎么评价**:
- 👍 16.9K stars / 2K forks（半年增长）；MIT 协议；中英双 README + 微信群 + Discord，国际化做得细

**风险与争议**: **学术 paperware 气质**——作者 Chao Huang（@huang_chao4969）个人推广驱动（5K→7.6K→8.3K stars 阶段性营销），Rohan Paul、@shao__meng 等 KOL 转发形成扩散，但 Reddit/HN 等工程社区独立讨论稀少。依赖链重（LightRAG + MinerU + VLM）且 setup 门槛高于一般 RAG。论文数据可信度遭社区挑战后无公开回应。

**技术备注**: Python 3.10；MIT；基于 HKUDS/LightRAG（图 RAG）；解析层 MinerU 或 Docling；VLM 层支持 OpenAI/Claude/Ollama；PyPI 包名 `raganything`

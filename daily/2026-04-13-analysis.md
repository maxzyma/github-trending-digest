### [microsoft/markitdown](https://github.com/microsoft/markitdown) — NEW ⭐ 104,904（+2,513）| Python | 信源：强

**它是什么**: 微软出品的"万能文档转 Markdown"工具——把 Office 文档（Word/Excel/PowerPoint）、PDF、图片、音频等各种格式，统一转成 Markdown 纯文本。一行命令搞定，不用装一堆不同的解析库。已有官方 MCP Server，可直接接入 Claude Desktop。

**解决什么问题**: 搭建 AI RAG 系统时，最头疼的是"喂数据"——你有上千份 Word、Excel、PDF，但 LLM 只吃纯文本。以前得用好几个不同的库分别处理，现在一个 MarkItDown 全搞定。相比 Unstructured 更简单，相比 LlamaParse 完全开源。

**谁在用、怎么用**:
- LLM/RAG 管道的文档预处理（最主流用法）
- HN 主帖 329 points / 81 comments——社区讨论与 Pandoc 的区别（Pandoc 更通用，MarkItDown 更聚焦 AI pipeline）
- MCP Server 直接接入 Claude/Cursor，Office 文件即时转换
- 医院/企业遗留 .doc 文件批量迁移（#37, #153）
- #1263（17 评论）请求页级 PDF 提取——说明用户需要细粒度控制
- #114（14 评论）中文图片转换失败——非英文场景短板

**社区怎么评价**:
- 👍 HN 讨论认可度高，r/Python 和 r/LocalLLaMA 用户称"比 Unstructured 简单"
- 👍 插件机制已上线，社区开始实验自定义 converter
- 👎 **Obsidian 创始人 kepano 公开批评**："grown into a big messy project"——认为微软应提供原生 Office→Markdown 高质量库，而非套壳 wrapper
- 👎 **本质是 wrapper**：底层依赖 mammoth/pandas/pdfminer 等第三方库，非微软自研
- 👎 纯图片 PDF 输出空白无报错（Discussion #1361）——静默失败最危险
- 👎 563 open issues，核心贡献者仅 2 人（afourney 100 + gagb 70 commits 占 77%）
- 📊 104,904 stars / 375 watchers（280:1，极高）；6,558 forks；563 open issues

**风险与争议**: 104K stars 的明星项目但核心贡献者仅 2 人，563 open issues 处理不过来。PDF 处理是用户最强刚需但恰是最大短板——竞品 Docling（IBM）在复杂 PDF 上更强（表格、数学公式、阅读顺序）。kepano 的批评直击要害：微软最了解 Office 格式，却用第三方库套壳，质量天花板受限。

**技术备注**: Python；MIT；wrapper 架构（mammoth/pandas/pdfminer）；MCP Server 可用；插件机制

### [rustfs/rustfs](https://github.com/rustfs/rustfs) — NEW ⭐ 25,262（+182）| Rust | 信源：中

**它是什么**: 一个用 Rust 写的对象存储系统——和 MinIO 做同一件事（S3 兼容的文件存储），但声称在小文件场景下快 2.3 倍。可以理解为"Rust 版的 MinIO"。定位 MinIO 的 drop-in replacement（二进制直接替换）。

**解决什么问题**: MinIO 是 Go 写的，在高并发小文件（如 AI 训练数据、日志碎片）场景下性能有瓶颈。更关键的是 MinIO 改用 AGPLv3 许可证，商用受限。RustFS 使用 Apache-2.0，对企业更友好。

**谁在用、怎么用**:
- Homelab S3 后端：Nextcloud/Immich/restic 备份（最普遍场景）
- Synology NAS Docker 部署（教程驱动）
- K8s 集群对象存储（Hetzner 用户实例）
- Milvus 官方博客评估其作为向量数据库的 S3 后端
- HN 8 条讨论（最高 38 points）——"Migrate from MinIO via binary replacement"引发实际迁移讨论
- #1553（29 评论）：S3 兼容性不完整，"probably not perfectly compatible with MinIO"
- #1298（32 评论）：不支持 MinIO DirectPV 创建的 StorageClass

**社区怎么评价**:
- 👍 Apache-2.0 对标 MinIO AGPLv3——许可证是核心卖点
- 👍 r/rust 评价"promising"；Proxmox 论坛有 homelab 用户好评
- 👎 **CVE-2025-68926（CVSS 9.8）**：gRPC 硬编码 token `"rustfs rpc"`，所有部署共享同一密码，允许未认证攻击者擦除存储数据——**严重安全事件**（已在 alpha.78 修复）
- 👎 **官方明确声明"Do NOT use in production"**——仍在 alpha 阶段
- 👎 S3 API 兼容性不完整（#1553）；erasure coding 不稳定（#89）
- 👎 Stars/Watchers 比 377:1（极端偏高），25K stars 但仅 67 watchers
- 📊 25,262 stars / 67 watchers；1,076 forks；92 open issues；核心 3 人占 75% commits

**风险与争议**: CVSS 9.8 的安全漏洞（硬编码 token）暴露代码审查流程严重缺失——对于存储系统这是致命信号。官方自己说不要用于生产。Benchmark "2.3x faster" 仅限 4KB 对象，HN 社区建议"benchmark your own workload"。Stars/Watchers 比 377:1 极端异常，增长模式存疑。

**技术备注**: Rust；Apache-2.0（对标 MinIO AGPLv3）；alpha 阶段；S3 兼容；CVE-2025-68926 已修复

**谁在用、怎么用**:

**风险与争议**: Brendan Mulligan 的判断最关键——Ralph 是过渡产品，Cursor/Claude Code 原生集成此模式只是时间问题。作者已转向 Antfarm（更完整的 agent 团队方案），ralph 可能被事实上放弃。停更 2 个月 + 63 个 open issue 无人处理证实了这一趋势。

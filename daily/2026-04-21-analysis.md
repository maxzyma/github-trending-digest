### [pi-hole/pi-hole](https://github.com/pi-hole/pi-hole) — NEW ⭐ ~57,209（+196）| Shell | 信源：强

**它是什么**: 网络级广告黑洞——装在树莓派或任何 Linux 机器上，充当你家/公司的 DNS 服务器，在域名解析层面拦截广告和追踪请求。智能电视广告、手机 App 追踪、IoT 设备遥测，只要走 DNS 就能拦。12 年历史（2014 年至今），57K stars，是自托管广告拦截的事实标准。

**解决什么问题**: 浏览器装 uBlock Origin 只保护浏览器，手机 App、智能电视、IoT 设备的广告和追踪无处可挡。Pi-hole 在网络层面解决——所有设备的 DNS 请求都经过它，一台设备保护全网。有用户发现家庭 15-20% 的网络流量是追踪请求。

**谁在用、怎么用**:
- 家庭用户：树莓派 + Pi-hole = 全家广告拦截（r/pihole 社区标配）
- 企业/小团队：DNS 级追踪防护
- 技术爱好者：Pi-hole + Unbound = 递归 DNS + 广告拦截（HN 849 分讨论）
- Docker 部署越来越普及（#6391 讨论 Docker 日志问题）

**社区怎么评价**:
- 👍 HN 多次登顶（849/810/587/573 分），"家庭网络必装"级别
- 👍 45 open issues / 57K stars，维护极为健康；262+ 贡献者
- 👎 HN v6 发布帖（573 分/304 评论）评价两极——"缺 feature"、"开发 4-5 年发布公告只有 6 个要点"
- 👎 #5635（65 评论）高磁盘 I/O 导致 DNS 解析失败，SD 卡场景真实痛点，长期未解决
- 👎 Reddit 常见对比：Pi-hole 适合 tinkerer，AdGuard Home 适合小白、功能更现代（DoH/DoT 原生）
- 📊 v6 重写后持续迭代至 v6.1.3/v6.6；EUPL 许可（非标准，商业集成需法务审查）

**风险与争议**: AdGuard Home 竞争加剧——原生 DoH/DoT、家长控制、更友好的 UI。EUPL 许可非主流，可能影响商业集成。v6 重写耗时 4-5 年暴露团队规模有限的问题。今日上榜属于"经典项目周期性回潮"。

**技术备注**: Shell + C（FTL engine）；EUPL-1.2；DNS sinkhole；支持 Docker/裸机/树莓派；Gravity 数据库；Web UI（AdminLTE）

### [XTLS/Xray-core](https://github.com/XTLS/Xray-core) — NEW ⭐ ~37,472（+78）| Go | 信源：中

**它是什么**: V2Ray 的社区分叉和继任者——一个多协议网络代理平台，核心创新是 REALITY 协议：无需自有域名即可伪装成合法 TLS 流量，绕过深度包检测（DPI）。在中国、俄罗斯、伊朗等网络审查严格的地区，这是数百万用户"翻墙"的核心基础设施之一。

**解决什么问题**: 传统 VPN 协议（PPTP/L2TP/OpenVPN）的流量特征早被 GFW 识别。Xray 的 VLESS + REALITY 组合让代理流量看起来和访问正常 HTTPS 网站一模一样——DPI 设备无法区分你在翻墙还是在正常浏览网页。

**谁在用、怎么用**:
- 个人用户：配合 3X-UI/Marzban 面板搭建代理服务器
- 路由器用户：OpenWrt/PassWall 透明代理（pfSense/MikroTik 社区强烈要求原生集成）
- 机场运营者：多用户管理 + 流量统计

**社区怎么评价**:
- 👍 REALITY 协议公认领先，性能测试可达直连 1.5 倍速
- 👍 发布节奏稳定（v26.4.17，2026-04-17 最新）
- 👎 #4952（304 评论）移除 VMess 协议引发巨大争议
- 👎 #3884（414 评论）Web Panel 安全清单引爆社区治理争论
- 👎 #4878（39 评论）VLESS+Vision+Reality 偶发 SSL 错误未解
- 👎 多位用户公开表示从 Xray 切换到 sing-box（@choicky），理由：协议覆盖更全、配置统一
- 📊 37K stars / 400 watchers；核心贡献者 RPRX(405)+yuhan6665(210)+Fangliding(131) 前三占 75%，bus factor 偏低

**风险与争议**: sing-box 竞争压力明显——新项目倾向选择 sing-box，Xray 优势在存量部署。社区治理争议频发（VMess 移除、安全清单、MMDF 事件均为数百评论级争吵）。项目主要服务反审查场景，有 GitHub 封号风险（已有开发者因用 Actions 验证 VPN 节点被封）。HN 和 Reddit 几乎无讨论，传播主要在中文技术社区。

**技术备注**: Go；MPL-2.0；VLESS/VMess/Trojan/Shadowsocks 多协议；REALITY/XTLS-Vision；gRPC/WebSocket/HTTP/2 传输层

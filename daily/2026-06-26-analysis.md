### [alibaba/page-agent](https://github.com/alibaba/page-agent) — NEW ⭐ 19,863（+163）| TypeScript | 信源：强

**它是什么**: 一个"住在网页里"的 AI 操作助手。你给它一句自然语言（比如"把这张表单按上次的格式填好"），它就直接在当前页面上点击、输入、选择，替你完成多步 GUI 操作。和"在浏览器外面遥控网页"的方案不同，它跑在页面内部，几行代码就能嵌进自己的 Web 应用。

**解决什么问题**: 过去要给 SaaS 加 AI copilot，往往要改后端、接多模态视觉模型、成本高。page-agent 用纯文本 DOM 操作（基于 browser-use 思路），省 token、响应快、无需后端改造，把"20 步点击"压成"一句话"。

**谁在用、怎么用**（来自 Issues/HN）:
- SaaS 内嵌 copilot：开发者几行代码集成到自家产品（HN 讨论核心场景）。
- ERP/CRM 智能填表、无障碍语音操作：issue #129 期待 Chrome 扩展、#222 请求 hover action，说明有人在真实场景里用并提需求。
- 痛点反馈：#290 选不中 antd 下拉框、#467 识别不出 input、#112 同一动作循环调用——纯 DOM 方案对复杂组件库适配仍不稳。

**社区怎么评价**:
- 正面: HN Show HN「A GUI agent that lives inside your web app」147 分 / 76 评论，讨论 "inside-out" 范式价值；多家英文技术博客转载。
- 负面: 复杂 UI 组件识别失败、动作循环，早期阶段生产可用性待验证。
- 健康度: stars/watchers ≈ 337:1（大厂明星项目结构性高比值），forks 1,707（fork:star ≈ 1:11，正常使用型），open issues 48，阿里团队活跃维护，建仓 2025-09-23（约 9 个月）。

**风险与争议**:
- **stability**: 复杂组件库（antd 下拉、动态 input）识别失败、动作循环，早期阶段。
- **bus_factor**: 阿里团队主导，贡献者集中度偏高。
- **vanity**: 337:1 高比值，但 HN 高质量讨论托底，单凭比值不足判虚高。

**技术备注**: TypeScript / MIT，纯文本 DOM 操作（非视觉多模态），可几行代码嵌入现有 Web 应用。

- **dependency**: 文档解析质量依赖底层模型与 OCR 能力，复杂版面仍有边界。

**技术备注**: Python，输出 markdown/JSON，面向 agentic 文档摄入。作为 OpenDataLab 维护的成熟基础设施记录为 tracker 首入。

## 如何理解今天的热度信号

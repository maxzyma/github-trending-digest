### [ogulcancelik/herdr](https://github.com/ogulcancelik/herdr) — NEW ⭐ 9,047（+486）| Rust | 信源：强

**它是什么**: 一个"住在终端里"的 AI agent 多路复用器。你可以在一个界面里同时开好几个编码 agent（Claude Code、Codex、Copilot 等），一个重构、一个写测试、一个查 bug，侧栏统一显示每个 agent 的状态，不用来回切窗口。

**解决什么问题**: 并行跑多个 coding agent 时，人得在一堆终端标签/tmux pane 之间来回跳，还容易忘了哪个卡住了、哪个在等输入。herdr 把它们收进一个带状态感知的面板（🔴blocked/🟡working），并支持 SSH 远程接入、断连保活、甚至手机接入。

**谁在用、怎么用**（来自 Issues/HN）:
- HN 用户 `shepherdjerred` 常开 5-8 个 agent 并行，`zihotki` 并行 4 个会话，都靠 herdr 的面板统一盯状态。
- 跨平台使用中暴露真实打磨期 bug：macOS nix build 失败（#830，7 评）、OSC 11 深浅色背景查询错误（#714，6 评）、macOS 输入法切换错乱（#774，4 评）。

**社区怎么评价**:
- 正面: Show HN 161 分/101 评，`ricardobeat`"用了几周取代十几个终端窗口"、`nullbio`"比 tmux 好用很多"；卖点 pane 状态感知获认可，且非 mac-only 开源对比 conductor.build/emdash 加分。
- 负面: `scubbo`"试了两天，conductor.build 更好，跑终端里只是炫技"；多人质疑"tmux 不够用吗""真需要并行这么多 agent"。
- 健康度: 9,047 ⭐ / 22 subscribers（411:1，龄短爆发）、创建约 3 个月、30+ 贡献者但作者 ogulcancelik 848 commits 绝对主导、open issues 30、昨日仍活跃提交。

**风险与争议**:
- **bus_factor**: 单一作者绝对主导（848 vs 次高 41 commits）。
- **stability**: 早期打磨期，多个跨平台 build/终端兼容 bug。
- **vanity**: 411:1 比值高，但有 HN 高质量讨论 + 真实 fork/贡献支撑，需按龄短解读，非典型空心化。
- **需求存疑**: HN 多人质疑 tmux 可替代性，"是否真需要并行这么多 agent"未有定论。

**技术备注**: Rust 编写，官方集成十余种 coding agent，可跑在 sandbox 内，SSH+手机远程接入。

**社区怎么评价**:

- 负面: 部署 Python 版本/依赖解析脆弱，生产可靠性存疑。
- 健康度: 4,225 ⭐ / 25 subscribers（169:1）、创建约 3 个月、贡献者仅 3 人（sushicw 21 commits 主导）、HN 仅 2 条官方性质提交 0 评论、X 多为官方/KOL 放大（且常与另一工具 gws Google Workspace CLI 混淆）、Reddit 无讨论。

**风险与争议**:
- **dependency**: deploy 依赖解析脆弱（litellm/Python 版本 #36/#42）。
- **compliance**: 深度绑定 Google Cloud Agent Platform + ADK，vendor lock-in 明确。
- **vanity**: 169:1 + 传播靠官方/KOL、HN/Reddit 零独立讨论、仓龄短、贡献者 3 人，官方营销强于自然采用。

**技术备注**: Python，围绕 Google ADK + Google Cloud 部署链路。

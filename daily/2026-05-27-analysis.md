### [DigitalPlatDev/FreeDomain](https://github.com/DigitalPlatDev/FreeDomain) — NEW ⭐ 167,480（+1,219）| HTML | 信源：中

**它是什么**：一个"给所有人免费域名"的服务，由非营利组织 DigitalPlat（Hack Foundation 501c3 资助）运营。你可以申请 `.dpdns.org` / `.us.kg` / `.qzz.io` 等非标准后缀的域名（技术上是它的二级域名），配上 Cloudflare DNS，挂到 GitHub Pages / Vercel 上，全程免费。这个 GitHub 仓库是它的"门面 + 申请入口"。

**解决什么问题**：曾经免费的 Freenom（.tk/.ml 等）已倒闭，个人开发者、学生想要一个不花钱的域名做小项目就没了去处。FreeDomain 接了这个缺口——号称已发放 50 万+ 域名。

**谁在用、怎么用**（来自 Discussions / 第三方）:
- 场景 1：学生 / 个人开发者通过外部门户 domain.digitalplat.org 申请域名，挂个人主页或小项目（非商业）
- 场景 2：社区支持走 GitHub Discussions（163 条），但多为"域名删不掉 / SSL 卡在 Pending Validation / 印度手机号无法注册"的求助帖
- 场景 3：**star 本仓库可多领一个免费域名**——把 starring 直接做成了申请工作流的一步

**社区怎么评价**:
- 👍 正面：填补 Freenom 空缺、有正规非营利背书（Hack Foundation）、对学生友好
- 👎 负面：Discussions 标题直白的 "All DigitalPlat domains are Spamhaus DBL listed"、"Service outage?"；LowEndTalk 质疑可持续性（"哪天一封邮件说不干了，域名就没了"）、"免费子域名你并不真正拥有"
- 📊 健康度：167,480★ / 213 subscribers（**786:1**）/ 3,092 forks / **Issues 已禁用**（API 报的 33 open_issues 是陈旧假象）/ 贡献者仅 2 人（EdwardLab 72 commits + 1 人 1 commit）

**风险与争议**:
- **vanity**: 786:1 是数字仓观测以来绝对新高，但根因是 **star-to-apply 机制**（star 即可多领一个域名）—— star 数与真实关注彻底脱钩，是"机制性刷星"而非代码价值
- **compliance**: us.kg 因被钓鱼 / 诈骗团伙大量滥用，被吉尔吉斯注册局**整体暂停解析**，被迫迁移 dpdns.org；新域名同样进 Spamhaus DBL，邮件几乎发不出；`paypal-secure.dpdns.org` 式钓鱼是结构性风险
- **bus_factor**: 单人维护（创始人 Edward Hsing / EdwardLab），官方 Telegram 账号曾被盗已弃用
- **stability**: 反复 service outage + SSL "Pending Validation" 卡死 + KYC 注册门槛
- **dependency**: 非标 TLD 受第三方注册局政策约束（us.kg 暂停即先例），用户对域名无所有权

**技术备注**：HTML / 申请门户 domain.digitalplat.org / Cloudflare DNS 后端 / 非标 TLD（dpdns.org / us.kg / qzz.io 等）。

### [st-tech/ppf-contact-solver](https://github.com/st-tech/ppf-contact-solver) — NEW ⭐ 3,507（+170）| Python | 信源：强

**它是什么**：一个物理仿真用的"接触求解器"——专门算布料、软体、绳索这类可形变物体之间碰到一起时怎么互相挤压、不穿透。做出来的衣服褶皱、软体碰撞会很真实、不会"穿模"。来自日本时尚电商巨头 ZOZO 的研究团队。

**解决什么问题**：传统物理仿真里，物体碰撞容易出现"穿模"（互相穿透）或布料像橡皮筋一样被拉伸失真。ppf-contact-solver 用 barrier（势垒）方法保证 penetration-free（绝不穿透）+ strain-limited（布料不被过度拉伸），仿真质量更高。

**谁在用、怎么用**（来自 HN/Issues/Blender 社区）:
- 场景 1：ZOZO 自身的虚拟试衣 / 数字服装布料仿真延伸应用
- 场景 2：VFX / Blender 离线动画——官方提供 Blender 插件（macOS 走远程引擎）
- 场景 3：学术界复现接触仿真论文（有 `sigasia-2024` 分支锁定论文版本）

**社区怎么评价**:
- 👍 正面：penetration-free 接触 + strain-limited 布料质量获认可；研究本身在 HN 被肯定；作者（论文一作）亲自回复 issue，维护活跃
- 👎 负面：HN 主帖 80 分 41 评，核心批评是"标题党"——Two Minute Papers 视频叫《游戏里最糟的 bug 永远消失了》，但实测 `46.4s/frame`、`3.38 min/frame` 是 **offline 离线、非实时**，对游戏运行时无用；#30 指文档被 LLM 引入冗余；Windows Native 部署踩坑多（#9 CUDA `named symbol not found`、#20/#21 参数报错）
- 📊 健康度：3,507★ / 32 subscribers（110:1，但由 TMP 视频带量解释）/ 251 forks / 11 open issues / commits 几乎全由 Ryoichi Ando（论文一作）提交，近一周仍每日推进

**风险与争议**:
- **CUDA-only**: 强制 NVIDIA + CUDA 12.8 + x86，推荐 RTX 4090/5090，arm64 / macOS 本地不支持——硬件门槛高
- **bus_factor**: 单作者 Ryoichi Ando 独占提交，是最大隐患
- **离线非实时**: 病毒视频造成"游戏可用"的错觉，实际是离线仿真，star 暴涨与真实工程采用脱节（增长主因是 TMP 视频 + Blender 插件，而非新论文）
- **真实产出反例标尺**: 与今日 vanity 样本相反，这是"真论文 + organic 病毒 + HN 实质辩论"的健康样本

**技术备注**：Python / CUDA / barrier-based（IPC 系）接触法 / ACM TOG SIGGRAPH Asia 2024 论文 / 提供 Blender 插件 / 来自 ZOZO Research（日本）。

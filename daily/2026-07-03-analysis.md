## NEW 项目

### [JuliusBrussee/caveman](https://github.com/JuliusBrussee/caveman) — NEW ⭐ 81,053（+926）| JavaScript | 信源：强

**它是什么**: 一个 Claude Code / Codex 通用「技能包」，核心卖点是"像原始人一样说话（why use many token when few token do trick）"来压缩 agent 的输出、省 token。装上后用 `/caveman` 切换 6 档压缩强度（lite/full/ultra + 三档文言文风格），并配套 caveman-commit（生成简短 commit）、caveman-review、caveman-compress（压缩 CLAUDE.md 输入约 46%）。

**解决什么问题**: Claude Code / Codex 长会话里模型输出啰嗦，token 账单快速累积。caveman 用「电报体/原始人体」指令让模型少废话，宣称砍掉 65% 输出 token。对高频重复改代码、批量重构的用户是直接的省钱手段。

**谁在用、怎么用**（来自 Issues/HN/评测）:
- 场景 1: 常规改代码、重构时开启压缩省 token；评测共识是复杂调试要关掉（模型需要逐步推理，压缩会伤害质量）。
- 场景 2: 跨 23+ 种 agent CLI 安装使用，但安装脚本脆弱——OpenCode 报 ENOENT 缺 caveman-compress.md（#426）、PowerShell 7 安装报错（#366）、不兼容 Antigravity CLI（#428）。
- 场景 3: 用户请求持久化模式（#141）、ChatGPT 集成（#131）、AXON notation 支持（#53），显示多平台真实试用。

**社区怎么评价**:
- 正面: 传播极广，多篇独立评测（andrew.ooo、skillsllm、mayhemcode）、Product Hunt 条目、404 Media 报道成本议题；常规编码场景确有省钱效果。
- 负面: HN 第三方 benchmark《I benchmarked caveman against "be brief."》（89 分 / 67 评）质疑——相对一句 "be brief" 到底有多少增量价值。
- 健康度: 81,053 stars / 183 subscribers ≈ **443:1**（极高）；open issues 359；20 贡献者但代码高度集中在作者一人；2026-04-04 创建，不到 3 个月冲到 8 万星，仍活跃维护。

**风险与争议**:
- **vanity**: 443:1 比值 + 三个月 8 万星 + 大量评测导流，传播热度远超社区 grounding。
- **误导性指标**: "65%" 仅指 output tokens，而 output 通常只占 Claude Code 账单 10–30%，独立评测算真实账单省约 5–15%（甚至 ~4%），核心主张被语境夸大。
- **stability**: 跨 23+ agent 的安装脚本脆弱，多个安装 issue 未闭合。

**技术备注**: JavaScript，纯 prompt/skill 层，无模型改动；跨 Claude Code / Codex / OpenCode / Cursor 等多 agent host 分发。

### [agentskills/agentskills](https://github.com/agentskills/agentskills) — NEW ⭐ 21,634（+86）| Python | 信源：强

**它是什么**: Agent Skills 的规范与文档仓——不是可执行工具，而是定义「一个 agent 技能长什么样」的开放标准：SKILL.md 用 YAML frontmatter + markdown 指令 + scripts/references/assets 组织，靠 progressive disclosure 省 context（每技能启动约 30–50 token）。规范由 Anthropic 于 2025-12 发起并开放，现已从 anthropics/skills 迁出、托管在中立组织 agentskills 下。

**解决什么问题**: 各家 agent（Claude Code、Codex、Gemini CLI、Cursor、Copilot）各自定义技能格式会导致碎片化。agentskills 想做跨平台的技能事实标准，让一份 SKILL.md 能在多个 agent host 上复用。

**谁在用、怎么用**（来自 Issues/HN）:
- 场景 1: 各 agent host 按 spec 实现技能加载；仓库本身是治理/RFC 场所——#15（118 评）讨论标准技能文件夹位置、#254/#255（.well-known URI 发现分发）、#81（npm/JS 包分发 RFC）。
- 场景 2: 开发者按 spec 编写可跨平台复用的技能包，被 OpenAI Codex、Gemini CLI、Cursor、Copilot 采用。

**社区怎么评价**:
- 正面: HN 多次高分——"Agent Skills" 544 分/260 评、376 分/212 评、"SkillsBench" 364 分；Firecrawl、Strapi 等多篇独立解读；纳入 Linux Foundation 旗下 Agentic AI Foundation 治理（与 MCP、AGENTS.md 同框）。
- 负面: 批判性讨论也真实——"You're probably using Agent Skills wrong"（74 分）、安全侧"skills 成为攻击面""87% 仍降低 agent 安全性"。
- 健康度: 21,634 stars / 183 subscribers ≈ 118:1（文档/规范类仓的典型特征，大量围观少量深关注）；open issues 47；forks 1,375；2025-12-16 创建，持续更新。

**风险与争议**:
- **安全面**: skills 被反复点名为 agent 的攻击面（扫描器绕过、指令注入）。
- **碎片化**: 多个核心 RFC 未定（技能位置、分发机制），采用者各自扩展 spec 存在分裂风险。
- **bus_factor**: org 归属中立，但治理透明度依赖 Linux Foundation 与少数核心维护者。

**技术备注**: 规范仓，代码 Apache 2.0 + 文档 CC-BY-4.0；对标 MCP 定位为 agent 生态的跨厂标准层。

### [openai/codex-plugin-cc](https://github.com/openai/codex-plugin-cc) — NEW ⭐ 22,655（+352）| JavaScript | 信源：中

**它是什么**: OpenAI 官方发布的 Claude Code 插件，让你在 Claude Code 里通过 slash 命令直接调用本地 Codex CLI——`/codex:review` 常规审查、`/codex:adversarial-review` 对抗审查、`/codex:rescue` 委托 Codex 做二次实现/调试、`/codex:transfer` 把整段会话迁到 Codex 续跑。定位是「Claude 为主、Codex 当第二审查者 / 后台 worker」的双模型互补。

**解决什么问题**: 开发者常想让第二个模型对 Claude 的产出做独立复核或接手卡住的任务，过去要手动切工具、复制上下文。这个插件把 Codex 变成 Claude Code 会话内可直接调度的审查者与救援 worker。

**谁在用、怎么用**（来自 Issues/博客）:
- 场景 1: 在 Claude Code 内用 `/codex:adversarial-review` 让 Codex 对抗式审查 Claude 写的代码，作为提交前的第二道关。
- 场景 2: 卡壳时用 `/codex:rescue` 把任务委托给 Codex 做深度根因排查或另一套实现；issue #346/#343/#328/#129 集中在 rescue 的 completion signal、broker 状态清理、多轮对抗审查等生命周期硬化。

**社区怎么评价**:
- 正面: 话题性强——媒体以「竞品互调（When Rivals Collaborate）」切入，多篇上手指南；OpenAI 官方社区有发布贴。
- 负面: HN 讨论热度弱（发布日最高仅 8 分 1 评）；Reddit 无有效独立讨论；缺第三方规模化实战反馈。
- 健康度: 22,655 stars / 72 subscribers ≈ 314:1（OpenAI 官方 org 结构性偏高，不宜单凭比值判 vanity）；open issues 265；贡献者集中在 dkundel-openai；2026-03-30 创建，已到 v1.0.4。

**风险与争议**:
- **cost**: 官方明确警告 review gate 的 Stop hook 会造成 Claude/Codex 长循环，快速耗尽 Codex 用量额度。
- **dependency**: 依赖本地 Codex CLI + ChatGPT 订阅/API key，同机双工具链。
- **stability**: 三个月龄、v1.0.x，rescue 生命周期仍在硬化中。

**技术备注**: JavaScript，Claude Code 插件形态；桥接 OpenAI Codex CLI，是「大厂官方跨模型互操作」的代表性样本。

### anthropics/claude-code — NEW ⭐（+8,764）| Shell | 信源：强

**它是什么**: Anthropic 官方的 AI 编程工具——在终端里运行，理解你的整个代码库，通过自然语言帮你写代码、解释代码、执行 Git 操作、跑测试。不是插件或扩展，而是独立的命令行工具，直接在你的项目目录里工作。

**解决什么问题**: 程序员日常有大量重复性工作（写样板代码、调试、提交代码、写文档）。Claude Code 让你用自然语言描述需求，AI 直接在你的终端里执行——读文件、改代码、跑命令，一气呵成。

**谁在用、怎么用**: 我们自己就是重度用户。全球开发者社区广泛采用，已形成庞大的技能（Skills）和插件生态。今天榜单上多个项目（claude-howto、oh-my-claudecode、superpowers 等）都是围绕它构建的。

**社区怎么评价**:
- 👍 今日 +8,764 远超所有其它项目，说明受到重大事件驱动（可能是新版本发布或重大功能更新）
- 👍 已形成平台级生态——本周 trending 上至少 6 个项目直接服务于 Claude Code
- 📊 Anthropic 官方维护，不存在维护风险

**风险与争议**: 无。官方项目，活跃维护。

**技术备注**: Shell + TypeScript；需要 Anthropic API key

### openai/codex — NEW ⭐ 72.4K（+1,416）| Rust | 信源：强

**它是什么**: OpenAI 官方的终端编程助手——和 Claude Code 类似，在终端里运行，用自然语言帮你写代码、改文件、执行命令。用 Rust 写成，沙盒化执行，是 OpenAI 对 Claude Code 的正面回应。

**解决什么问题**: 开发者希望在命令行环境中直接用 AI 辅助编码，不想离开终端切到网页或 IDE 插件。

**谁在用、怎么用**:

- 远程开发需求强烈（#10450，589 赞）——无法连接 SSH 远程主机，阻碍了大量专业开发者采用
- macOS Intel 用户（#10410，286 赞）——桌面版仅支持 Apple Silicon

**社区怎么评价**:
- 👍 HN 516 分 / 289 评论，热度很高
- 👎 **Token 消耗过快**（#14593，374 评论，150 赞）——用户反映 2 小时烧掉 20% 配额，这是最大不满
- 👎 不支持 macOS Intel（#10410，286 赞）
- 👎 安全担忧——有 ANSI 转义码注入漏洞报告
- 📊 72.4K stars / 2,297 open issues（积压严重）

**风险与争议**: Token 成本不透明是核心痛点。2,297 open issues 消化速度是问号。本质上是 OpenAI API 的终端前端，受限于 API 定价和速率限制。

**技术备注**: Rust；Apache-2.0

### google-research/timesfm — NEW ⭐ 12.9K（+1,195）| Python | 信源：强

**它是什么**: Google 研究院开发的"时间序列预测基础模型"（200M 参数）——给它一段历史数据（股价、气温、用电量等随时间变化的数据），它能预测未来走势。类似 GPT 对文本的作用，但对象是时间序列数据。无需针对每个场景单独训练，开箱即用。

**解决什么问题**: 传统时间序列预测需要针对每种数据单独调参建模，耗时且需要专业知识。TimesFM 用一个预训练模型覆盖多种场景（需求预测、能源负荷、金融指标等），大幅降低门槛。

**谁在用、怎么用**:
- 量化金融、能源需求预测、零售库存规划
- 但实际使用反馈偏少——HN 讨论更多是对方向的兴趣，而非实际应用

**社区怎么评价**:
- 👍 HN **317 分 / 118 评论** 和 **315 分 / 108 评论** 两篇，关注度很高
- 👍 Google Research 背书
- 👎 **安装是最大痛点**——排名最高的 3 个 issue 全是安装问题：praxis/lingvo 依赖冲突（#1，48 评论）、conda 失败（#24）、fine-tuning 未打包到 PyPI（#242）
- 👎 依赖 Google 内部生态（JAX/Flax/Praxis/Lingvo），在非 Google 环境安装极其痛苦
- 📊 12.9K stars / 181 open issues；维护团队回复 issue 较慢

**风险与争议**: **工程化打磨严重不足**——从论文到可用产品的距离还很远。安装门槛阻碍了实际采用。

**技术备注**: Python + JAX/Flax/PyTorch；200M 参数；Apache-2.0

### zai-org/GLM-OCR — NEW ⭐ 5.1K（+238）| Python | 信源：强

**它是什么**: 基于大语言模型的 OCR 工具——不只是"认字"，还能理解文档的版面布局。把图片/PDF 中的文字、表格、公式提取成结构化文本（Markdown/JSON）。与百度 PaddleOCR（昨日上榜）形成直接竞争，差异化在于用多模态大模型一步到位解决文字识别+版面理解+信息抽取。

**解决什么问题**: 传统 OCR 对复杂文档（混合排版、表格嵌套、手写体）识别率低，需要多个工具串联。GLM-OCR 用一个模型搞定——适用于票据、合同、学术论文等场景。

**谁在用、怎么用**:
- 票据解析、文档抽取——#67（8 评论）vLLM 部署时大图报错，说明有人在实际业务中部署
- 信息抽取（KIE）任务——#169（5 评论）vLLM 做 KIE 时出现幻觉问题

**社区怎么评价**:
- 👍 HN **302 分 / 75 评论**，关注度高
- 👎 vLLM 部署兼容性问题密集——大图报错、配置缺字段崩溃、KIE 幻觉
- 📊 5.1K stars / 46 open issues；项目不到 2 个月，非常年轻

**风险与争议**: 项目极新（2 个月）；维护者 zai-org 是新组织，长期维护能力待观察；生产环境落地门槛不低（vLLM 部署问题多）。

**技术备注**: Python + GLM 多模态模型 + vLLM；Apache-2.0

### axios/axios — NEW ⭐（+216）| JavaScript | 信源：强

**它是什么**: JavaScript 世界最流行的 HTTP 请求库——几乎所有前端项目和 Node.js 后端项目都用它来调 API。每周 npm 下载量数千万。

**解决什么问题**: 浏览器原生的 `fetch` API 功能简陋（不支持请求拦截、超时、自动重试等）。axios 提供了更完善的 HTTP 客户端体验。

**谁在用、怎么用**: 全球数百万前端/后端 JavaScript 开发者。知名老牌库，今日增量低（+216），短暂上榜。

**技术备注**: JavaScript；MIT

### f/prompts.chat — NEW ⭐ 156.6K（+1,014）| HTML | 信源：强

**它是什么**: 前身是"Awesome ChatGPT Prompts"（GitHub 全站 Top 级别项目，156K stars）——一个社区驱动的 AI 提示词集合。收录了各种角色扮演、写作助手、编程辅助等场景的提示词模板。现已升级为 prompts.chat，从 Markdown 列表变成带 Web 界面的 Next.js 应用，支持企业自托管。

**解决什么问题**: 不知道怎么写好提示词来驱动 AI。提供现成模板——"充当 Linux 终端"、"充当英语翻译"、"充当面试官"等，降低使用门槛。

**谁在用、怎么用**:
- 企业自托管——#1056 请求 OIDC/OAuth 认证插件，说明有团队在部署

**社区怎么评价**:
- 👍 HN 182 分 / 51 评论，中等热度
- 👍 156.6K stars / 1,609 watchers，GitHub 顶级项目
- 👎 大量 star 有"收藏不用"的虚高成分
- 👎 prompt 内容随模型迭代快速贬值，长期价值存疑
- 📊 CC0 许可证——内容无版权保护，质量参差不齐

**风险与争议**: 从 Markdown 列表转型为 Web 应用可能导致老用户流失。重命名 + 自托管升级可能是本次上榜的驱动因素。

**技术备注**: Next.js + HTML；CC0 许可证

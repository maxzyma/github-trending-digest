**主线一：榜单结构性两极分化——vanity 膨胀的 agent/skill 仓占据头部，真有机老项目沉到腰部。** 18 个项目里 stars:subscribers（subscribers = 真实 watch 数）> 100:1 的有 14 个（78%），延续了 6/05–6/06 创纪录的 vanity 密度。但今天出现一个清晰的**结构对照**：榜首聚集的是 mvanhorn/last30days-skill（255:1）、MemPalace（173:1）、Agent-Reach（315:1）、CopilotKit（199:1）、obra/superpowers（260:1，219K stars 绝对值惊人）这类 AI agent/skill 仓；而 sveltejs/svelte（93:1）、nginx/nginx（32:1）、golang/go（40:1）、openai/whisper（137:1）、vitejs/vite（156:1）这五个真有机的基础软件老项目，今天**首次进入本 tracker**，却挤在第 7–15 名、日增 stars 只有 20–30。注意：whisper/vite/trivy/PaddleOCR 的高 ratio 不是 vanity——超大热门仓的 watch 行为本就稀缺，ratio 在这里会失真；真正的 vanity 信号是「高 ratio + 零社区足迹 + 仓龄短 + 从未上榜却已巨量 stars」的组合。

**主线二：santifer/career-ops 是今日最干净的 vanity-inflation 样本——2 个月仓龄、49K stars、HN/Reddit/X 三大公开社区零有机讨论。** 这个"基于 Claude Code 的 AI 求职系统"created 于 2026-04-04，仓龄仅约 2 个月，commit 总数约 221（作者一人 99 个、第二名 13 个），却已累积 49,385 stars / 仅 205 subscribers（241:1）。issue 区只有 13 个低热度工程 bug，HN Algolia 仅 2 条 story（各 2 分 0 评论从未上首页），Reddit/X 一手帖均无法定位。唯一外部声量来自 ProductHunt + 作者自营博客（career-ops.org/blog、santifer.io）+ 二手聚合站——全部是作者可控/营销渠道。star 增速与 community footprint 严重背离，是继 5/26 affaan-m/ECC（4 个月 19 万 stars + HN/Reddit 双零）之后又一个教科书级 vanity 样本。

**主线三：AI agent 安全/隔离赛道首次以大厂官方姿态登榜——microsoft/mxc 把"agent 沙箱"做成 OS 级强制隔离层。** mxc（Microsoft eXecution Container）是 Build 2026 首发的 Rust 项目，用声明式 JSON policy 定义 AI agent 能访问的文件/网络/剪贴板/UI，由 Windows 内核运行时**强制**（非事后捕获）。启动伙伴名单含 OpenAI（Codex）、NVIDIA、GitHub Copilot CLI、Manus、Nous Research、OpenClaw——基本是当前 agent 生态的半壁。这与同在榜上的各类 agent/skill 框架形成赛道闭环：当 agent 能"看整个互联网"（Agent-Reach）、能跑任意 skill（superpowers/PAI）时，"怎么把不可信 agent 关进笼子"成了大厂级刚需。官方明确声明"当前任何 MXC profile 都不应视为安全边界"（early preview），热度来自发布会与一线媒体而非草根验证。

### [danielmiessler/Personal_AI_Infrastructure](https://github.com/danielmiessler/Personal_AI_Infrastructure) — NEW ⭐ 14,977（+70）| TypeScript | 信源：中

**它是什么**: 一套以 Claude Code 为运行时的"个人 AI 基础设施（PAI）"——把你的目标、记忆、习惯、自定义技能组织成一个常驻的"生活操作系统"，让 AI agent 替你持续追踪和执行。作者 Daniel Miessler 是知名安全人物（Fabric 框架作者）。

**解决什么问题**: 之前个人用 AI 是一次性对话，关掉窗口就忘了你是谁、在做什么。PAI 给 AI 装上持久记忆（你的 TELOS 目标体系）、自定义 skills、一个叫 Pulse 的后台守护进程 + 仪表盘，让 AI 成为长期对齐你个人目标的助手，而不是每次从零开始。

**谁在用、怎么用**（来自 GitHub Discussions/Issues）:
- 场景 1: 个人开发者把它当"模板"改造自己的 Life OS——社区共识是"PAI 是 pattern 不是 product，要 template 别 fork"。
- 场景 2: 想把 PAI 装到 OpenCode 等其他 runtime 的用户受阻（#1300，5 评论），因为它深度绑定 Claude Code。

**社区怎么评价**:
- 👍 正面: security-first 架构受认可（对比 OpenClaw 暴露端口的做法更克制）。
- 👎 负面: vendor lock-in 焦虑公开化——#1258（7 评论，最热）担忧 Anthropic usage policy 变更下 `claude -p` 的计费问题；多个 Discussion（#542）质疑"号称 vendor-agnostic 实则重度绑定 Claude Code"，请求 Ollama/本地模型支持。
- 📊 健康度: 14,977 stars / 205 subscribers（73:1，偏高但未到极端）；208 open issues / closed PR 仅 1 个 → 28 contributors 实为 Daniel 近乎单人维护（他自述"是 one-person effort，也是生计"）；仓龄仅 9 个月。

**风险与争议**:
- **vanity**: 知名安全 KOL（Fabric 作者）+ Unsupervised Learning newsletter + Cognitive Revolution 播客两次专访 + LinkedIn 联动，9 个月 15K stars / 单人维护 = 典型 audience-driven 受众导入型增长，非自然工程口碑；HN/Reddit/X 三大公开社区零讨论，反馈仅在自有渠道回声室。
- **dependency**: vendor lock-in 争议公开化（#1258/#1300/#542），号称 agnostic 实则重绑 Claude Code，受 Anthropic 计费政策直接影响。
- **bus_factor**: 208 open issues / 1 closed PR，协作能力薄弱，几乎纯作者单人维护。

**技术备注**: TypeScript；以 Claude Code 为 runtime；核心组件 TELOS（目标体系）、Pulse daemon、自定义 skills。

**它是什么**: 一个跑在 Claude Code 上的本地 AI 求职 CLI 工具——把岗位 URL 喂进去，AI 给岗位打 A–F 分（候选人侧反向筛选公司），生成定制简历 PDF，再用 tracker 跟进投递状态。明确拒绝 auto-apply（不自动海投）。

**技术备注**: JavaScript + Go dashboard；跑在 Claude Code 上；14 skill 模式 + PDF 生成 + 批处理；本地 CLI 而非托管服务。

### [microsoft/mxc](https://github.com/microsoft/mxc) — NEW ⭐ 592（+64）| Rust | 信源：强

**它是什么**: Microsoft eXecution Container——专为 AI agent 设计的操作系统级沙箱。用一份声明式 JSON policy 写清楚 agent 能碰哪些文件、能访问哪些网络、能不能读剪贴板/操作 UI，然后由系统内核**强制执行**这套规则（不是事后审计，而是事前拦截）。

**解决什么问题**: 之前把 AI agent 从 demo 推向生产时，没有可靠手段限制"agent 跑起来后到底能动什么"——一个被注入的 agent 可能读你全盘文件或外联。mxc 提供一个"可组合的沙箱光谱"：从轻量进程隔离（给编码 agent）到 micro-VM 硬件级隔离（给敏感负载），跨 Windows/Linux/macOS 多 backend（ProcessContainer / Bubblewrap / Seatbelt / microVM / Hyperlight）。

**谁在用、怎么用**（来自官方 + 媒体）:
- 场景 1: 企业把 AI agent 投产的隔离层。已公布启动伙伴：OpenAI（Codex）、NVIDIA（OpenShell）、GitHub Copilot CLI（已生产用 process isolation）、Manus、Nous Research、OpenClaw。
- 场景 2: 7 月起与 Agent 365 叠加 Entra/Intune/Defender/Purview 做中心化治理 + 合规审计（应对 EU AI Act 高风险场景）。

**社区怎么评价**:
- 👍 正面: VentureBeat 称其为"微软 Build 今年最具影响力的平台动作"；启动伙伴名单覆盖 agent 生态半壁。
- 👎 负面: HN 无讨论（仅 2 条 self-post 各 2–3 分 0 评论），Reddit/X 无讨论——草根开发者口碑尚未形成。
- 📊 健康度: 592 stars / 5 subscribers，18 open issues；created 2026-02-06、Build 2026（6/2–3）首发，仓龄约 4 个月仍在 push；确为微软官方仓（org=microsoft，MIT，npm `@microsoft/mxc-sdk`，schema 0.6.0-alpha）。

**风险与争议**:
- **stability**: 官方明确声明"当前任何 MXC profile 都不应视为安全边界"，early preview，已知 policy 过度宽松待修；schema alpha、多数 backend experimental。
- **vanity**: 热度来自发布会与一线媒体（VentureBeat/SC Media/Visual Studio Magazine）而非草根验证，真实开发者口碑缺位（118:1 ratio 是大厂官方发布的关注落差，非长尾刷星）。

**技术备注**: Rust（+ TypeScript SDK）；声明式 JSON policy + 内核运行时强制；多 backend 跨平台；定位 agent 安全/隔离赛道。

## 首次上榜项目（NEW）

### [heygen-com/hyperframes](https://github.com/heygen-com/hyperframes) — NEW ⭐ 30,051（+395）| TypeScript | 信源：中

**它是什么**: 一个"用 HTML 写视频"的开源框架。你像写网页一样写 HTML/CSS，它用无头 Chrome 逐帧渲染再用 FFmpeg 合成成 MP4 视频。卖点是"AI agent 天生会写 HTML，所以让视频格式 = HTML，agent 就能零成本变成视频编辑器"。背后是数字人公司 HeyGen。

**解决什么问题**: 在它之前，程序化做视频主流是 Remotion（用 React 写视频），对 AI agent 不够友好——agent 要先理解 React 组件模型。Hyperframes 把视频降维成纯 HTML，让 Claude Code / Cursor / Codex 等编码 agent 直接生成可渲染的视频脚本，`npx hyperframes init/preview/render` 一条命令出片。

**谁在用、怎么用**（来自 Issues/评测）:
- 配合编码 agent 生成营销/演示视频：写 HTML → preview → render MP4，常对标 Remotion（纯 HTML vs React 路线）。
- HeyGen 自身生产环境在用；第三方评测提及 tldraw、TanStack 等团队尝试。
- Issues 偏务实功能/性能反馈：#337（最热，6 评）问如何接 ElevenLabs TTS、#1661 请求浏览器端导出、#1653 报告 0.7.0 init 开销致约 2x 性能回归。

**社区怎么评价**:
- 正面: andrew.ooo、blog.nidhin.dev 等多篇技术评测正面，认可"video as code"思路；issue 围绕真实使用，无负面骂战。
- 负面: HN 多个 Show HN 帖但最高仅 6 分、评论 0–2；X/Reddit 无独立讨论；已有性能回归报告。
- 健康度: 30,051 stars / 82 subscribers ≈ 366:1（比值极高）；但 forks 2,825（fork/star ≈ 9.4%，远高于普通项目，说明确有大量实操 clone）；open issues 59；贡献者 10 人但前 3 名均为 HeyGen 员工，厂商主导；3.5 月龄，高频维护（push 2026-06-23）。

**风险与争议**:
- **bus_factor**: 贡献几乎全来自 HeyGen 员工，独立社区贡献薄。
- **vanity**: 366:1 高比值 + HN 低分，传播热度强于独立 grounding；但高 fork 率与扎实 issue 缓解此判断。
- **dependency**: 依赖无头 Chrome BeginFrame API + FFmpeg。
- **stability**: 项目仅 3.5 月，已现性能回归（#1653），未完全成熟。

**技术备注**: TypeScript；CLI `npx hyperframes`，无头 Chrome 逐帧渲染 + FFmpeg 合成；Apache-2.0；与 Remotion 同赛道不同路线。

### [JCodesMore/ai-website-cloner-template](https://github.com/JCodesMore/ai-website-cloner-template) — NEW ⭐ 17,798（+100）| TypeScript | 信源：中

**它是什么**: 一个"用 AI 一条命令克隆任意网站"的项目模板。你点 GitHub 的"Use this template"复制一份，指向目标网址，跑 `/clone-website`，AI 编码 agent 会截图、提取设计 token、再并行把页面重建成一个干净的 Next.js 16 + shadcn/ui 工程。

**解决什么问题**: 在它之前，"照着某网站做一个类似的"要么人肉抠 CSS、要么用截图工具只能得到静态图。这个模板强调"克隆行为而非截图"——让 agent 理解页面结构并产出可维护的现代前端代码，支持 13 种 agent（推荐 Claude Code Opus）。

**谁在用、怎么用**（来自 Issues）:
- 模板式工作流：复制模板 → 指向 URL → `/clone-website` → agent 截图/提取 design token/并行重建为 Next.js 工程。
- Issues 有真实用户反馈与功能诉求：#49 用户称"best cloner I've found, leaves code super clean"并追问 hybrid 路线图、#18 请求 Playwright MCP 集成、#3 Figma/Google Stitch MCP 支持。
- #48 提到需手动 bump Next.js 16.2.7 修依赖漏洞（安全维护滞后信号）。

**社区怎么评价**:
- 正面: issue #49 真实好评（代码干净）；功能诉求活跃，说明有实际使用者。
- 负面: HN 直接搜索 0 命中、X/Reddit 无可直接引用的独立讨论；传播主要靠 trendshift 榜单引流；近三周（自 2026-06-01）未更新。
- 健康度: 17,798 stars / 90 subscribers ≈ 198:1；forks 2,748——但这是 template repo，"Use this template"复制模式天然推高 fork，不能直接当空心化铁证；open issues 仅 12；作者 JCodesMore 独占 35 次提交，其余 5 人各 1 次，贡献高度集中。

**风险与争议**:
- **vanity**: 198:1 比 + HN/X/Reddit 几乎零有机讨论 + 仅 3 月龄 + 靠榜单引流；但高 fork 需按 template 模式修正解读。
- **bus_factor**: 作者独占贡献，近三周停更。
- **compliance**: 克隆任意网站存在版权/ToS 风险。
- **dependency**: 依赖 13 种 AI agent + MCP，安全更新需手动 PR 推动（#48）。

**技术备注**: TypeScript；GitHub template repo，产出 Next.js 16 + shadcn/ui；支持 Claude Code/Cursor 等 13 种 agent。

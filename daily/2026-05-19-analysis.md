### [NVlabs/Sana](https://github.com/NVlabs/Sana) — NEW ⭐ 6,548 (+387) | Python | 信源：强

**它是什么**：NVIDIA Research 出品的高效高分辨率图像合成模型套件，核心是 Linear Diffusion Transformer（线性注意力 DiT），**已发 6 个模型系列**：SANA 原版 / 1.5 / Sprint（蒸馏加速）/ Video / WM（World Model 视频）/ Sol-RL（post-training），全部在 HuggingFace 上有权重。包含训练 + 推理 + 4-bit/8-bit 量化 + ComfyUI 节点 + diffusers 集成 + SGLang serving 全栈代码。已发 ICLR 2025/2026 双 Oral 论文。

**解决什么问题**：在消费级 GPU（8GB VRAM）上跑高质量 1024x1024 + 视频生成，**线性注意力比传统 quadratic attention 显存与速度更优**，是开源 image/video diffusion 阵营对标 Stable Diffusion 3 / FLUX / Sora 的主力。

**谁在用怎么用**：学术圈采纳为主——arXiv 上多个新论文（SLA、Gated Condition Injection 等）把 SANA 列为 linear-attention DiT 基线。社区集成项目（ComfyUI workflow、diffusers pipeline、8GB VRAM 推理脚本）是工程用户主要入口。issue #203 "How do we use the 4.8b model?"（28 条评论）是 1.5 4.8B 模型使用文档不清的实证；issue #227 ComfyUI workflow 失败（16 条）是工程接入痛点。

**社区怎么评价**：**2026-05-16 SANA-WM 登上 HN 首页 399 分/152 评论**（"SANA-WM, a 2.6B open-source world model for 1-minute 720p video"），5-17/18 MarkTechPost 等媒体跟进。该事件直接触发 5/19 trending 上榜——**完整事件链可追溯**：5/16 PR #374 SANA-WM 合并 → HN 首页 → 媒体扩散 → trending。

**风险与争议**:
- **bus_factor**: lawrence-cj（Junsong Chen, NVIDIA）116 commits 占绝大多数，其余 contributor 主要是 yujincheng08 (20) + xieenze (NVIDIA 论文一作 13) + lyttttt3333 (Sol-RL 作者 3)，离职/转岗会停摆

- **hype 与权重 gap**: SANA-WM 在 HN/媒体爆红时 `inference_sana_wm.py` 尚未提交（#377/#372 实锤"are Models available or not?"），作者私下回复"2-3 天后发"——**X/媒体投放节奏快于内部 release 节奏**，与 5/14 NVIDIA-AI-Blueprints 同款样本
- **commercial 限制**: 代码 Apache-2.0，但模型权重 HuggingFace 上多为 CC-BY-NC，**商业受限**

**技术备注**：与 5/14 `NVIDIA-AI-Blueprints/video-search-and-summarization` 同公司不同性质——NVlabs = NVIDIA Research（学术影响力 KPI），AI-Blueprints = NVIDIA 企业 SI（卖 GPU/Enterprise license KPI），用户群、维护团队、KPI 都不同，本周双双冲榜是巧合而非协同推广。NVIDIA 一公司在 trending 上展示了**两类完全不同的开源策略投影**：科研论文 repo 走 HN 算法触发，企业蓝图 repo 走 PR 稿 + 矩阵号触发。

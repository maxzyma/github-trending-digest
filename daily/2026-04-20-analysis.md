### [ruvnet/RuView](https://github.com/ruvnet/RuView) — NEW ⭐ ~（+149）| Rust | 信源：弱

**它是什么**: 一个用 WiFi 信号"隔墙感知人体"的平台。原理：每个 WiFi 路由器都在发射无线电波，人在房间里移动、呼吸甚至静坐时都会干扰这些波。RuView 用低成本 ESP32 传感器捕获 WiFi 信道状态信息（CSI），通过信号处理和神经网络把这些干扰转化为：人体存在检测、呼吸/心率监测、活动识别、姿态估计（17 个 COCO 关键点）。无需摄像头、无需穿戴设备、无需互联网。基于 Carnegie Mellon 的 DensePose From WiFi 研究。

**解决什么问题**: 传统人体感知需要摄像头（隐私问题）或穿戴设备（使用门槛）。WiFi 感知用已有的无线电基础设施，穿墙、暗光、无隐私侵入地实现存在检测和生命体征监测——适用于养老看护、智能家居、安防等场景。

**谁在用、怎么用**:
- 智能家居：房间占用检测、入睡监测
- 养老看护：非接触式呼吸/心率监测、跌倒检测
- 安防：穿墙存在检测
- ESP32 节点低至 $9/个

**社区怎么评价**:
- 👍 技术含金量极高——43 个 ADR、15 个 Rust crate、1,031+ 测试
- 👍 完全边缘计算，无云依赖
- 👎 Beta 阶段，ESP32-C3/原版不支持
- 👎 单 ESP32 空间分辨率有限，需要多节点或 Cognitum Seed
- 📊 项目极其硬核，受众狭窄

**风险与争议**: WiFi 感知的精度远不如摄像头或毫米波雷达，实际部署中的环境干扰（家具、多人、宠物）是巨大挑战。"隔墙看人"的能力也引发隐私和安全担忧。Beta 阶段，API 和固件可能随时变更。

**技术备注**: Rust + Python + C（ESP32 固件）；MIT；ESP32-S3/S2/C6 硬件；CSI 信号处理 + ONNX 推理；43 个 ADR；1,031+ 测试

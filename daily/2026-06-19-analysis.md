**主线二：今日最扎实的真信号在开源模型发版。** GLM-5/GLM-5.2 是 grounding 最厚的一项——HN 多帖高分（初发 484 分、GLM-5.2「Is Out」768 分、登 Artificial Analysis 开源权重榜首 883 分），SWE-bench Pro 62.1 超 GPT-5.5、成本约 1/6，MIT 开放权重 + 昇腾全栈训练；LTX-2 是 Lightricks 的真厂商发版（19B 音视频联合生成、单卡 4090 可跑、第三方评测 image-to-video top-3）。这两项的热度有真实技术验证和外部评测支撑，是今日最值得点开的项目；iroh 1.0 余热延续也属真事件驱动。

**主线三：榜单解读提醒 — AI 编码/agent 工具多为短龄高比值，营销驱动需谨慎。** 这一块多数项目 stars/subscribers 比值偏高且社区足迹薄：palmier-pro 209:1（YC S24 + X 营销主导，HN 近乎零讨论）、agent-native subscribers 仅 2（官方/作者自推，HN/Reddit 零独立讨论）、codebase-memory-mcp 187:1、OpenMontage 105:1 且疑似 6 周停更。它们的概念新颖，但热度主要由 YC/公司/KOL 投放驱动，真实独立采用证据少，建议结合 Issues/HN/Reddit 足迹判断，不要把 Trending 排名等同于落地使用。

### [zai-org/GLM-5](https://github.com/zai-org/GLM-5) — NEW ⭐ 4,372（+478）| — | 信源：强

**它是什么**: Z.ai（原智谱 AI）的旗舰开源大模型系列。GLM-5 初代 2026-02-11 发布，744B MoE（每 token 激活约 40B）、200K 上下文、28.5T token 预训练、MIT 许可开放权重，并用华为昇腾等国产芯片完成训推；当前 repo 已迭代到 GLM-5.2（753B、1M 上下文、6-16 发布）。定位「从 Vibe Coding 到 Agentic Engineering」，专攻长周期自主编码与系统工程。

**解决什么问题**: 为需要长周期、全仓库自主编码的 agent 提供一个高能力、可商用（MIT）、成本远低于闭源前沿模型的开放权重选项。官方数据 SWE-bench Pro 62.1（超 GPT-5.5 的 58.6）、FrontierSWE 74.4%（逼近 Claude Opus 4.8），成本约为 GPT-5.5 的 1/6。

**谁在用、怎么用**（来自 Issues/HN/Reddit）:
- 已上 HuggingFace + Z.ai API + 20 多个第三方编码环境（OpenCode、Codex 等），企业订阅 $12.6/月起。
- 工具链集成诉求活跃：#39 请求 OpenAI `/responses` 兼容以适配 Codex；#15 经 NVIDIA NIM 的 tool-call JSON 畸形导致 OpenCode 解析失败。

**社区怎么评价**:
- 正面: HN 多帖高分（768/883 分），登 Artificial Analysis 开源权重榜首；r/LocalLLaMA 整体正面，看重 SWE/数学分数 + MIT 商用 + 昇腾全栈训练。
- 负面: repo 内 issue 多为可用性投诉——#83「GLM-5.2 API 因严重限流连续 2 天近乎瘫痪」、#68「无法使用 API」；r/singularity 指 ARC-AGI 2 偏弱，部分用户质疑实测对不上跑分。
- 健康度: 4,372 stars / 31 subscribers ≈ 141:1（偏高，但旗舰模型 repo 结构性偏高，需谨慎解读，非空心化）；forks 447、open issues 34；2026-02 创建仍活跃。

**风险与争议**:
- **stability**: API 限流/宕机有多个 issue 实证（#83/#68）。
- **dependency**: NVIDIA NIM 路径 tool-call 解析存在缺陷（#15）。

**技术备注**: MoE 架构 + DeepSeek/IndexShare 稀疏注意力，MIT 许可；arXiv GLM-5 技术报告（2602.15763）。

### [Lightricks/LTX-2](https://github.com/Lightricks/LTX-2) — NEW ⭐ 7,589（+196）| Python | 信源：强

**它是什么**: Lightricks 开源的音视频联合生成基础模型（19B 参数：14B 视频 + 5B 音频），主打"单次推理同步生成视频 + 音频"，免去音频后期工序。能力上支持原生 4K、50 FPS、最长 20 秒，text-to-video / image-to-video 双模，且可本地跑（单张 RTX 4090/5090、16GB+ VRAM，经 ComfyUI 或桌面编辑器）。2026-01 CES 开放权重发布，3 月发 LTX-2.3。

**解决什么问题**: 大多数视频生成模型只出画面，配音/音效要另做后期。LTX-2 把视频和音频放进一次推理，并把硬件门槛压到消费级单卡，让独立创作者本地生成带声音的短片。

**谁在用、怎么用**（来自 Issues/HN/评测）:
- 独立创作者/初创做 4K 带音短片、本地化生成，集成进 Fal/Replicate/ComfyUI 的二创生态（类 Flux/SD 路线）。
- HN「Show HN: Audio-to-Video with LTX-2」提到 foley 音效效果好，但难区分相似声音。

**社区怎么评价**:
- 正面: Artificial Analysis 将其 image-to-video 排进 top-3（次于 Kling 3.5、Veo 3.1）；forks 1,214 较高，本地可跑受欢迎。
- 负面: #121（13 评论）推理张量维度报错（8192 vs 2048）；#148 LTX-2.3 视频结尾出现伪影；#180 训练显存门槛过高；#142 文档链接错误。
- 健康度: 7,589 stars / 79 subscribers ≈ 96:1（接近警戒线，但属知名厂商真发版，需谨慎解读，非 vanity）；open issues 94，2026-01 创建、6-17 仍活跃维护。

**风险与争议**:
- **compliance**: 营销页宣称 Apache 2.0，但 GitHub 实际 LICENSE 是「LTX-2 Community License」（ARR < $10M 免费商用，超过需购商业授权）；`license: Other` 印证非纯开源，商用以 GitHub LICENSE 为准。
- **stability**: 长视频时序稳定性、推理张量报错、结尾伪影（#121/#148）。
- **cost**: 训练显存门槛高（#180）。

**技术备注**: Python，权重经 HuggingFace 分发，ComfyUI/desktop editor 接入；许可证为自定义 Community License。

### [palmier-io/palmier-pro](https://github.com/palmier-io/palmier-pro) — NEW ⭐ 1,464（+749）| Swift | 信源：中

**它是什么**: YC S24 公司 Palmier 出品的开源 macOS 原生视频编辑器（Swift 从零写，对标 Premiere Pro）。核心卖点是"为 AI 而生"——通过本地 MCP server（`127.0.0.1:19789/mcp`）让 Claude/Codex/Cursor 直接在时间线上剪辑、重排、生成素材，并内置 Seedance/Kling/Nano Banana Pro 等生成模型。

**解决什么问题**: AI 视频制作常在工具间反复横跳（Runway 生成 → Premiere 导入 → 不满意再回去重生成）。palmier-pro 把编辑器与 AI agent 经 MCP 直连，让 agent 在时间线上直接操作，编辑器 + MCP 开源免费，生成式 AI 走 credits 付费。

**谁在用、怎么用**（来自 Issues/X）:
- 官方称已用它为 15+ YC 公司制作电影级 AI 宣发视频；导出支持 MP4/ProRes 及 Premiere/DaVinci 的 XML。
- issues 集中在平台兼容诉求：#14 支持更老 macOS、#21 Intel Mac、#20 Linux、#32 OpenRouter 集成。

**社区怎么评价**:
- 正面: 概念契合 AI 视频工作流，X 上编辑圈对"agent 真能操作时间线吗"有好奇。
- 负面: 强约束（要求 macOS 26 Tahoe + Apple Silicon）劝退部分用户；无独立第三方评测。
- 健康度: 1,464 stars / 7 subscribers ≈ 209:1（极高）；forks 166、open issues 13；2026-04 创建约 2 月龄；`htin1` 610 提交、次者仅 2 提交，单人主导。

**风险与争议**:
- **vanity**: 209:1 比值 + HN 近乎零讨论，热度主要靠 YC/X 营销驱动，与社区足迹不匹配。
- **bus_factor**: 单人主导提交。
- **cost**: 生成功能闭源、credits 付费。
- **stability**: 约 2 月龄新仓 + 强平台门槛（仅 macOS 26 Tahoe + Apple Silicon）。

**技术备注**: Swift，macOS 原生；本地 MCP server 暴露剪辑能力给 AI 编码 agent。

### [BuilderIO/agent-native](https://github.com/BuilderIO/agent-native) — NEW ⭐ 841（+172）| TypeScript | 信源：弱

**它是什么**: Builder.io（CEO Steve Sewell，Mitosis 作者）开源的"agent-native 应用"框架。核心理念是把 AI agent 原生嵌入应用本身——agent 能看到/修改屏幕内容、跳转页面，凡 UI 能做的 agent 都能做，反之亦然。提供跨 agent/UI/HTTP/CLI 的统一 actions、SQL 状态同步、runtime（工具/记忆/jobs/可观测性）及现成模板（日历、内容编辑器、PPT、分析面板）。

**解决什么问题**: 定位填补"有 UI 但 AI 弱的 SaaS"与"强 AI 但无 UI 的裸 agent"之间的空白，让开发者构建 agent 与 UI 双向对等操作的应用。

**谁在用、怎么用**（来自 PR/官方）:

**社区怎么评价**:

- 负面: HN/Reddit 零独立讨论，缺第三方评测。
- 健康度: 841 stars / 2 subscribers（异常低，疑 API 或项目设置）；forks 101；2026-03 创建约 3 月龄；steve8708 占 1129 提交绝对主导，PR 号已到 #1320，含大量 bot 自动化提交。

**风险与争议**:
- **vanity**: 热度由公司/作者营销驱动，HN/X/Reddit 零独立讨论，subscribers 极低。

**技术备注**: TypeScript，Drizzle/Nitro 后端，提供 runtime + 统一 actions 抽象。

**社区怎么评价**:

### [gastownhall/beads](https://github.com/gastownhall/beads) — NEW ⭐ 21,700（+152）| Go | 信源：强

**它是什么**: Steve Yegge（前 Sourcegraph、Google、Amazon 老兵，Stevey's Drunken Blog Rants 作者）写的命令行任务追踪器，专给 AI coding agent 用——给 agent 一个 graph 结构的"issue tracker"，让它知道"任务 A 阻塞任务 B，任务 C 准备就绪可以开工"，**取代 markdown todo list 的脏乱与 LLM 健忘**。CLI 命令是 `bd`（"beads"），通过 `bd ready` / `bd create` / `bd update --claim` 让 agent 自己管理任务依赖。

**解决什么问题**: 在 Beads 之前，coding agent 长任务时只有 markdown plan + 上下文窗口，agent 会忘 plan、错乱依赖、重复做已做事。Beads 把 plan 抽成有依赖图的数据库，agent 每轮 cold start 时 `bd ready` 拿一组就绪任务即可，**agent 的"工作记忆"从 chat history 迁移到外部图数据库**。这套设计被 Anthropic 看中——Claude Code 的 task system 官方致谢"灵感来自 Beads"。

**谁在用、怎么用**（来自 Issues/X 真实场景）:
- **接入方式**: 在项目根加 `Use 'bd' for task tracking` 进 `AGENTS.md` / `CLAUDE.md`，agent 通过 shell 调用 `bd`；**不是 MCP server**，是 CLI + Claude Code plugin marketplace
- **集成范围**: Claude Code、Codex、Cursor、Gemini CLI、Cline、Amp 任意 agentic CLI 都能用
- **典型 stack**: 与 Ralph Wiggum loop（自驱循环）、MCP Agent Mail（agent 间消息）组成"2025 coding agent 生产栈"，Joe Heitzeberg、Jeffrey Emanuel、aary 等 X 上多次推荐
- **早期形态 → 当前形态**: 0.60 之前是 git-backed JSONL + SQLite，0.61+ 切到 Dolt（version-controlled SQL，支持 embedded / server 双模式）

**社区怎么评价**:
- 👍 正面: HN 2025-11-28 "Beads – A memory upgrade for your coding agent" **111 分 / 68 评论**（中等热度但讨论质量高）；X 上 Steve Yegge 本人多帖力推 + 多位生产用户称为 "drop-in cognitive upgrade"；Anthropic 将其概念产品化进 Claude Code（最强口碑背书）
- 👎 负面: [#2573 "Moving to dolt pretty much made beads unusable"](https://github.com/gastownhall/beads/issues/2573)（19 评论 / 29 +1）SQLite → Dolt 后任务消失、DB 无法访问；[#2559](https://github.com/gastownhall/beads/issues/2559) Dolt 重启后连接失败需 `bd init --force`；[#3135](https://github.com/gastownhall/beads/issues/3135) v0.60 还宣称 "git-backed"、v0.61 改 "powered by Dolt"，多 worktree 不同步；[#3059](https://github.com/gastownhall/beads/issues/3059) 仓库从 `steveyegge/*` 迁到 `gastownhall/*` 后 ~400 个旧引用未更新（go.mod、Claude Code plugin marketplace），用户卡 v0.60.0 无升级路径
- 📊 健康度: 21,701 stars / **82 真实订阅者** / 1,439 forks / 156 open issues / Top 1 commit Steve_Yegge / 创建 2025-10-12（约 6 个月）/ MIT

**风险与争议**:
- **被 Anthropic 蚕食**: Claude Code 自带 task system 后，Beads 在 Anthropic 用户群中价值减半；长期价值看能否扩散到 Cursor/Codex/Gemini 用户群
- **底座迁移裂缝**: Dolt 切换是 6 个月生死线——若用户继续吐槽 + 没有官方回退路径，会失去早期"git-backed"承诺的信任
- **治理疑点**: 仓库 2026 年初从 `steveyegge` 个人迁到 `gastownhall` org（同期创建），#3059 暴露迁移仓促

**技术备注**: Go / MIT / `bd` CLI / Dolt 数据库 / Claude Code plugin marketplace 内置；homepage gastownhall.ai

### [apernet/hysteria](https://github.com/apernet/hysteria) — NEW ⭐ 20,254（+51）| Go | 信源：中

**它是什么**: 一个用 QUIC 协议做"高速、抗封锁"网络代理的开源软件。简单说：在网络受审查或链路质量差（丢包、抖动）的环境里，能让你的浏览器/客户端到服务器之间的连接保持流畅。Hysteria 2 是目前中国"机场"（按量/按月卖代理流量的灰色 VPN 服务）的标配协议之一，伊朗/俄罗斯丢包链路也广泛使用。

**解决什么问题**: 传统 VPN 协议（OpenVPN/WireGuard/IPsec）在 GFW、伊朗/俄罗斯 DPI（深度包检测）面前要么直接被识别封锁，要么在丢包/抖动严重的链路上吞吐量崩塌。Hysteria 把 QUIC 的 0-RTT 握手 + 拥塞控制 + UDP 多路复用做了优化，在丢包率 30%+ 的环境下仍能维持 80%+ 吞吐——这是它相比 OpenVPN/sing-box 等通用栈的核心差异化。

**谁在用、怎么用**（基于 Issues + X + 第三方综述）:
- **场景 1（机场分发，最大用户基本盘）**: Hysteria2 + S-UI/3X-UI/Blitz 面板做用户管理 + 按量计费，是中国机场套餐主力协议。X 上 VilaVPN 等机场把"Hysteria 2 协议"做卖点对外营销。
- **场景 2（伊朗/俄罗斯丢包链路）**: 波斯语社区（kevinzakarian、iAghapour）大量教程推 Hysteria2 + S-UI 面板；Sailfish OS 论坛用户证言"Naive 和 Hysteria 2 在其他协议全挂时仍能工作"。
- **场景 3（个人翻墙）**: README 主打场景，Shadowrocket（iOS 客户端）官方做过 Hysteria 缓冲区修复——iOS 个人用户基本盘稳定。
- **不用**: 出海企业代理基本不用（企业方向是 sing-box + VLESS+Reality 组合），原因是 QUIC 的 UDP 特征 + 缺企业管理特性。

**社区怎么评价**:
- 👍 正面: 维护活跃（最新 release `app/v2.9.1` 发布于 2026-05-10，近 3 天连续 commit）；MIT 协议；机场生态自然传播——VilaVPN/Shadowrocket 等下游产品免费做品牌曝光。
- 👎 负面: [#1156](https://github.com/apernet/hysteria/issues/1156) "测速后无法连接 accepting stream failed: timeout"（34 评论，疑 GFW 干扰主战场）；[#1365](https://github.com/apernet/hysteria/issues/1365) Android 锁屏断连 10s 才恢复（27 评论）；[#340](https://github.com/apernet/hysteria/pull/340) Pluggable Transport 接口 PR 从 2022 挂到现在未合——抗封锁能力的可扩展接口长期 WIP；HN 仅 1 条 2023-10-26 92 分讨论，2 年无新热点。
- 📊 健康度: 20,254★ / **142 subscribers**（**0.7%**，stars/watchers = 143:1 偏低，反映用户多为"翻墙工具消费者"而非协议关注者）/ 2,097 forks / 237 open issues / MIT / 顶级贡献者 tobyxdd 977 commits + haruue 154，长尾稀疏。

**风险与争议**:
- **compliance**: 作者匿名（tobyxdd），项目定位即"censorship circumvention"，中国境内分发/部署属灰色地带，机场运营商被抓案例持续。
- **GFW 封锁周期**: 2025 年 GFW 开始对 QUIC 概率性丢包/封禁，Hysteria 的 UDP 特征比 TCP 更易识别，[#1156] 即典型受害症状。已被 VLESS+Reality（TCP + TLS 1.3 + 真实站点伪装）部分替代。
- **dependency**: 基于 quic-go，QUIC 协议层演进受上游约束；上游 RFC 变更可能引发协议特征改变。
- **bus_factor**: 核心维护几乎只靠 tobyxdd 一人（977/1,300+ commits），haruue 是唯一稳定二号位——抗封锁工具单点故障风险尤其敏感。

**技术备注**: Go + quic-go + MIT；服务端 + 多平台客户端（Linux/Windows/macOS/Android/iOS via Shadowrocket）；今日 +51 stars 是榜单倒数第二，"NEW"标记仅因首次进入本数字仓观测榜单（脚本对比昨日 PREV 文件），并非真正首次曝光——6 年项目长尾偶发上榜。

### [anonfaded/FadCam](https://github.com/anonfaded/FadCam) — NEW ⭐ 2,228（+116）| Java | 信源：中

**它是什么**: 一个开源、无广告的安卓摄影录像 App，支持后台录像（屏幕关掉/退到后台时仍在录）、屏幕录制、直播推流、远程相机控制。卖点是"隐私优先 + 本地存储"——录的视频不上传任何云。f-droid/IzzyOnDroid 商店分发，APK 直装；GPL-3.0。

**解决什么问题**: 普通安卓相机 App 关屏即停录、有广告、要求过多权限（Google Play 商店内的相机类 App 普遍权限索取激进）。FadCam 把"录像功能"做成纯本地工具——背景录像 + 自定义保存路径 + 不联网 + 可自定义通知栏图标（伪装成"News Update"等）。**第二层目标用户**：dashcam（行车记录仪）/ 家庭 CCTV / 个人证据采集等"长时间隐蔽录像"场景，这一层是 README "ethical use only" 免责声明真正服务的对象。

**谁在用、怎么用**（基于 Issues + Privacy Guides 论坛）:
- **场景 1（dashcam 行车记录仪）**: [PR #212](https://github.com/anonfaded/FadCam/pull/212) 社区贡献 Loop Recording 功能（满磁盘后循环覆盖），是行车记录仪场景明示需求。
- **场景 2（家庭 CCTV 远程监控）**: README 提及"remote camera control"，配合 Tailscale/局域网视频流。
- **场景 3（个人证据采集 + 取证）**: 国内"防偷拍取证"/"维权录像"等灰色用例隐含在功能矩阵中（黑屏伪装 Privacy Black Mode 即此场景）。

**社区怎么评价**:
- 👍 正面: f-droid / IzzyOnDroid 商店分发评分稳定；GitHub Discussions #20 等讨论真实场景需求积极；GPL-3.0 + 真正本地存储是隐私社区认可点。

- 📊 健康度: 2,228★ / **10 subscribers**（**0.45%**，stars/watchers = **223:1** 榜单极端值）/ 171 forks / 81 open issues / GPL-3.0 / 11 contributors，但 anonfaded 1,535 commits 占 99%+，第二位 weblate 翻译机器人 11 次，其余 ≤7 次——典型单人维护。

**风险与争议**:
- **vanity**: 223:1 stars/watchers 是本数字仓观测榜单极端值（前高 289:1 是 5/12 datawhalechina/easy-vibe）+ HN/X/Reddit 三大平台零讨论 + GitHub Trending +116 stars/天——爆发式涨星缺乏可验证的口碑承接，疑似集中流量推送（小众软件论坛 meta.appinn.net 中文圈有讨论）。

### [onyx-dot-app/onyx](https://github.com/onyx-dot-app/onyx) — NEW ⭐ 23.3K（+1,852）| Python | 信源：强

**它是什么**: 一个可以自己部署的"企业内部版 ChatGPT"——连接公司内部的 Slack、Confluence、Google Drive、Jira 等 50+ 数据源，员工用自然语言提问，AI 基于公司内部知识回答。支持深度研究、代码执行、图片生成、语音对话。前身叫 Danswer，YC W24 项目。

**解决什么问题**: 企业知识分散在各个系统中，员工找信息效率低。Onyx 通过 RAG 把所有内部知识统一索引，让 AI 有据可答而非凭空编造。同时支持多种大模型（OpenAI/Anthropic/本地 Ollama），不绑定供应商。

**谁在用、怎么用**:
- 活跃 PR 集中在连接器扩展（Jira SM、S3、GitLab、Bookstack）——**真实业务场景驱动**，用户在生产环境中使用并贡献代码
- Bounty 机制运作中（#9490，46 评论）——社区协作模式健康

**社区怎么评价**:
- 👍 HN Launch **254 分 / 160 评论**（2025-11）+ 前身 Danswer **231 分 / 129 评论**（2024-02）——两次 Launch 均高质量讨论
- 👍 YC W24 背书；23.3K stars / 3.1K forks；MIT
- 👎 改名后品牌辨识度差——"Onyx" 太通用，HN 搜索几乎无结果
- 📊 314 open issues（对此体量正常）；License 实际可能是开放核心模式（EE 部分另行许可）

**风险与争议**: 开放核心模式需确认 Enterprise Edition 的许可条款。品牌改名（Danswer → Onyx）后 SEO 是隐患。

**技术备注**: Python + RAG + 50+ 连接器；支持 OpenAI/Anthropic/Ollama；MIT（核心）

### [dmtrKovalenko/fff.nvim](https://github.com/dmtrKovalenko/fff.nvim) — NEW ⭐ 3.3K（+750）| Rust | 信源：强

**它是什么**: 一个"超级文件搜索引擎"——用 Rust 写成，同时服务两类用户：Neovim 开发者和 AI 编程助手（如 Claude Code）。核心卖点是快（10 万文件级别）、容错（打错字也能搜到）、带记忆（会学习你常用的文件，自动排序优先级）。

**解决什么问题**: AI 编程助手在大型代码库中找文件时，经常需要多次搜索、读很多无关文件，浪费时间和 token。fff.nvim 通过内置记忆（frecency、git 状态、文件大小等排序因素）让 AI 一次就找到正确文件。对人类用户则提供极强的拼写容错。

**谁在用、怎么用**:
- Neovim 用户用作文件查找器替代品（替代 telescope.nvim 等）
- AI Agent 通过 MCP 协议集成——作为 Claude Code 的文件搜索工具
- #72（22 评论）请求 ignore patterns——基础功能缺失但呼声高

**社区怎么评价**:
- 👍 Rust 实现，性能有保证
- 👎 **内存占用过高**（#285，18 评论）——大项目中可能不稳定
- 👎 **每次更新都要清本地数据**（#245，12 评论）——升级体验差，破坏性变更频繁
- 👎 MCP 安装方式是 `curl | bash`，存在供应链安全顾虑
- 📊 HN 66 分 / 15 评论，热度一般

**风险与争议**: 内存泄漏未解决；API 不稳定（频繁破坏性变更）；项目仍在快速迭代早期。

**技术备注**: Rust + Neovim 插件 + MCP server；MIT

**它是什么**: Google 官方的"手机端 AI 体验橱窗"——一个 Android/iOS 应用，让普通用户在手机上直接运行开源大语言模型（如 Gemma 4），无需联网无需云服务。功能包括 AI 聊天、图片识别、语音转写、Agent 技能，类似"端侧 AI 的 App Store"。

**解决什么问题**: 过去想在手机上跑 LLM 需要编译、配置环境，门槛很高。Gallery 让用户一键下载模型即可使用，同时为开发者提供参考实现。

- "Gemma 4 on iPhone" HN 帖获 414 分 / 112 评论——社区对端侧跑大模型兴趣极高

**社区怎么评价**:
- 👍 Google 官方出品；17K stars，增长极快
- 👍 HN 关联帖 414 分（Gemma 4 on iPhone），赛道热度高
- 👎 **GPU 兼容性问题严重**——前 3 个热门 issue 全是 GPU 崩溃/不稳定
- 👎 仍标注 "experimental Beta"；196 open issues 对展示型应用偏多
- 📊 竞品 Cactus（"Ollama for Smartphones"）HN 231 分，赛道拥挤

**风险与争议**: 端侧 GPU 推理稳定性是核心瓶颈，尤其跨设备兼容。Beta 阶段。

**技术备注**: Kotlin (Android) + Swift (iOS)；背后引擎是同组织的 LiteRT-LM

### [google-ai-edge/LiteRT-LM](https://github.com/google-ai-edge/LiteRT-LM) — NEW ⭐ 1.6K（+124）| C++ | 信源：强

**它是什么**: Gallery 背后的"发动机"——Google 开源的端侧大模型推理引擎，前身与 TensorFlow Lite 一脉相承。它是 Chrome、Chromebook Plus、Pixel Watch 等 Google 产品内部实际使用的端侧 LLM 运行时。支持 Android/iOS/Web/桌面/树莓派，提供 GPU 和 NPU 硬件加速。

**解决什么问题**: 模型训好了，怎么高效跑在用户设备上？LiteRT-LM 解决的是"最后一公里"部署问题——跨平台、多硬件加速、支持多模态和 function calling。

**谁在用、怎么用**:
- **Google 自己在生产环境使用**——README 明确写出用于 Chrome、Chromebook Plus、Pixel Watch
- #547（9 评论）Gemma3n 模型转换需求；#1038（6 评论）高通 NPU 信息缺失

**社区怎么评价**:
- 👍 Google 内部生产验证——最强的可信度信号
- 👍 CLI 工具 `uv tool install` 一键安装，开发者体验在改善
- 👎 HN 几乎无讨论（最高 6 分）——与 llama.cpp、MLX 相比知名度差距明显
- 📊 1.6K stars / 117 open issues；NPU 支持和模型转换是高频痛点

**风险与争议**: 作为底层基础设施，社区生态尚未形成。与 llama.cpp/MLX 竞争中知名度劣势明显。

**技术备注**: C++ 核心 + Python/JS/Swift 绑定；支持 GPU/NPU 硬件加速；Apache-2.0

### [badlogic/pi-mono](https://github.com/badlogic/pi-mono) — NEW ⭐ 31.9K（+355）| TypeScript | 信源：中

**它是什么**: libGDX 作者 Mario Zechner（badlogic）的"AI 全家桶"——一个人用 TypeScript 做的 monorepo，把编程 Agent CLI、统一多模型 LLM 接口、终端 UI、Web UI、Slack 机器人、GPU Pod 部署管理工具全部打包在一起。官网自嘲叫 shittycodingagent.ai。

**解决什么问题**: 接入不同 LLM 提供商（OpenAI/Anthropic/Google）需要各写一套适配；编程 Agent、Slack Bot、Web 聊天界面各自独立开发。Pi 把这些碎片统一成可组合的包，减少重复造轮子。

**谁在用、怎么用**:
- 热门 issue 围绕 Gemma 4 thinking 支持、prompt 模板参数提示——说明有人在实际使用
- SessionManager 阻塞异步持久化、WSL 剪贴板失效——生产环境 bug

**社区怎么评价**:
- 👍 libGDX 作者的个人声望；31.9K stars；MIT
- 👎 **HN 零口碑**——约 10 条提交全部零讨论，star 数与社区讨论度严重不匹配
- 👎 **单人项目**（bus factor = 1），作者会定期关闭 issue tracker 专注重构
- 📊 15 open issues（处理快）；Stars/forks 比合理

**风险与争议**: 单人维护 + HN 零口碑 + 自嘲域名，企业采纳门槛高。但作为个人项目的完成度和迭代速度令人印象深刻。

**技术备注**: TypeScript monorepo；统一 LLM API + 编程 Agent CLI + Slack Bot + vLLM Pod 管理；MIT

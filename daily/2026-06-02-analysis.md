### [nesquena/hermes-webui](https://github.com/nesquena/hermes-webui) — NEW ⭐ 11,382（+945）| Python | 信源：中偏强

**它是什么**: 一个让你从浏览器或手机上使用「Hermes Agent」的网页控制台。Hermes Agent 本身是 Nous Research 开源的一个自治编码/任务 agent（类似 Claude Code 或 OpenClaw，能持久记忆、定时任务、调用几十种工具），平时跑在你自己的服务器或电脑上、只能用命令行操作；这个项目给它套了一个三栏式网页界面，让你随时随地（通过 SSH 隧道 / Tailscale 连回自托管实例）发指令、看进度、批准危险操作。

**解决什么问题**: 没有它之前，自托管的 Hermes Agent 只能在终端里操作，离开电脑就用不了，也没法在手机上盯着长任务。有了它，编码 agent 变成一个随身可控的服务——多会话编排、子 agent 委派、危险命令审批都能在网页里点。

**谁在用、怎么用**（来自 Issues/HN）:
- 把 WebUI 当严肃生产工具的用户在推动架构演进：#1925「让 WebUI 变成 Hermes runtime 的薄观测/控制层」（19 评）、#1980 provider-neutral 的记忆审批队列、#1240 要求 Provider/Model 配置与 CLI/Gateway 单一真相源对齐。
- HN 上有人主动求「好用的 hermes web ui」（organic 需求佐证），也催生了十几个 piggyback 衍生项目（中文分支、一键 Docker 部署套件等）。

**社区怎么评价**:
- 👍 正面: 作者 @nesquena 高频自推并 @Nous Research / @Teknium 获官方与 KOL 转发；issue 区是架构性深度讨论而非灌水，工程态度认真。
- 👎 负面: 多为真实 bug——#3368/#3360 模型选择用 substring 匹配导致选错模型、#3346 旧消息去重失败、#3351 请求 Trusted Header 认证（安全短板）。
- 📊 健康度: 11.4K stars / 39 watchers ≈ 292:1（比例偏离健康区，但属结构性虚高）；forks 1,458、closed issues 850（解决率 85%）、open PR 64、贡献者 178、issue 编号已到 #3368——工程活跃度真实，非空心。

**风险与争议**:
- **vanity**: 292:1 star/watcher，star 增长高度依赖作者自推 + 紧贴 Nous Hermes 热点，HN/Reddit 第三方独立口碑缺失。

### [supermemoryai/supermemory](https://github.com/supermemoryai/supermemory) — NEW ⭐ 24,023（+647）| TypeScript | 信源：中

**它是什么**: 给 AI 应用和 agent 用的「记忆中枢」——一个能长期记住对话、文档、用户偏好的 API + 应用。你把信息喂给它，它自动抽取事实、处理矛盾、遗忘过期内容，下次 agent 需要上下文时低延迟取回。可以理解成「AI 时代的可托管长期记忆数据库」。

**解决什么问题**: LLM 本身没有跨会话记忆，每次对话都是白纸。没有它之前，开发者要自己拼 embedding + 向量检索 + 用户画像存储。supermemory 把这套打包成一个 API，主打「比 mem0/Letta 更低延迟」。

**谁在用、怎么用**（来自 Issues/HN）:
- 自称有真实付费客户（Cluely、Scira、Composio Rube、Montra），宣称在 LongMemEval/LoCoMo/ConvoMem 三大记忆基准上排第一。
- 起家于 2024 年的「chat with Twitter bookmarks」消费 app（曾达 5 万用户），后 pivot 为 agent universal memory API。

**社区怎么评价**:
- 👍 正面: 75 贡献者、每日 commit，开发极活跃；融资硬信源强（$2.6–3M 种子轮，投资人含 Jeff Dean、Cloudflare CTO 等 angel）。
- 👎 负面: 负面集中在核心检索路径——#792「Claude Code plugin 能写入但 search/recall 全返回空，API key 与 OAuth 双失效」（核心卖点可靠性问题）、#1027 `_retrieve_memories()` profile=None 崩溃、#1030 MCP 缺 listMemories。
- 📊 健康度: 24K stars / 93 watchers = 258:1；forks 2,150 偏高；open issues 仅 25 且多为团队自驱 PR（社区报 bug 少、内部迭代主导）。

**风险与争议**:
- **cost**: open-source repo 是商业云（supermemory.ai「Context Cloud」）的获客漏斗，core engine 逐步云端化，存在 lock-in 路径。
- **vanity**: 258:1 + HN 长期冷淡（历史帖普遍 1–5 分）vs 24K stars 反差，增长高度依赖创始人 X 个人光环 + 名人投资人背书。
- **stability**: #792 检索失效是结构性 bug，直接影响「低延迟记忆」核心卖点。

**技术备注**: TypeScript，MIT，知识图谱 + RAG + connectors 架构。竞品 mem0 / Letta / Memories.ai，差异化论点仅「更低延迟」，技术护城河叙事偏弱。

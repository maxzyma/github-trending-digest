## 本周趋势纵览

**趋势一：消费级开源视频剪辑「CapCut 替代」升为本周新赛道，OpenCut 是最直观的 C 端真信号。** 相较前几周榜单以 skills 与基础设施为主，本周多了一条贴近普通用户的主线：
- **OpenCut-app/OpenCut**（07-14 首入，66,588 → 74,050 stars）四天连续在榜，07-15 单日 +4,276 为全周单日增量最高，07-16 从 rank4 升至榜首。它是浏览器本地处理、无水印、无需登录、隐私优先的开源 CapCut 替代，直接命中 CapCut 出海版权/隐私顾虑下的替代需求。HN 主帖 447 分 / 151 评 + 近 7,000 forks 是强 grounding，不是纯关注度膨胀。
- **AIEraDev/Clypra**（07-15 首入，Tauri + React 桌面剪辑）同赛道跟进，定位重合（本地、多轨时间线、FFmpeg 导出），但成熟度差距明显——仓龄不足 3 个月、核心拖拽功能 #128 报坏、传播靠 KOL。

读者视角：关注「Web/桌面剪辑开源化」的可以放心点开 OpenCut（选型明显更稳），Clypra 更多是赛道热度的衍生跟风，选型前需核实可用性。需要注意 OpenCut 导出/预览核心能力仍在重构（#460 WebCodecs、#794 FFmpeg 回退），当前仅 Web 端相对可用。

- **工程侧**：mattpocock/skills（Matt Pocock 的 `.claude` 技能集，累计 174,327 stars，07-15~17 多日守前三）——回答「怎么写代码」。

- **OpenCut-app/OpenCut（NEW，66.6K→74K，TypeScript，信源强）**：本周最直观的 C 端真信号。浏览器本地剪辑、无水印、无登录、隐私优先的开源 CapCut 替代。HN 447 分 + 近 7,000 forks 是扎实 grounding。看点：导出/预览重构进度（#460 WebCodecs、#794 FFmpeg 回退）、桌面端何时成熟、超大 PR 治理争议（「128k line PR」HN 109 分）。当前仅 Web 端相对可用。

## 如何理解热度信号

本周把可信度结构分三档理解，方法与前几周一致：

1. **稳态基础设施/学习合集档（可放心）**：ossu/computer-science、build-your-own-x、awesome-llm-apps、spec-kit、penpot、PostHog、lobehub、Win11Debloat、openinterpreter（142:1 属 2023 老仓历史积累）。它们 stars/subscribers 比结构性偏高是正常的，属「老项目回归」而非炒作，不需要额外 grounding。
2. **强 grounding 真信号档（可信）**：OpenCut（forks 7K + HN 447）、Bonsai（HN 694 + r/LocalLLaMA 实测 + 多硬件 PR）、sharpemu（多论坛媒体独立报道 + 具体兼容性 issue）。这些项目 star 增长与社区讨论高度匹配，是本周点开价值最高的一档——但仍需分清「已跑通」与「仍在早期」（OpenCut 导出重构中、Bonsai 主流引擎不支持 1-bit、sharpemu 极早期 alpha）。
3. **短龄高失衡档（需谨慎）**：graphify（283→291:1、单人 800 提交、「省 70x token」出自营销博客未复现）、ui-skills（358:1、单人 156 提交、HN 5 分 0 评）、hallmark（360:1、KOL @nutlope 引流）、Clypra（265:1、核心功能报坏、靠 KOL）、grok2api（452:1、逆向 ToS 风险、全渠道零讨论）、exercises-dataset（342:1）。star 数不足以证明采用，需结合 issue 真实度、HN/Reddit/X 独立讨论、subscribers、仓龄综合判断。

一个重要的区分：**收藏/数据/教材型仓的高 ratio 不等于 vanity**。awesome-llm-apps（101:1）、exercises-dataset（342:1）、maths-cs-ai-compendium（84:1）的 star 表达的是「收藏/复用意愿」而非 watch 源码，subscribers 偏低是结构性正常，不应单独按 100:1 阈值判虚高。真正需要谨慎的是「短龄 + 全渠道无 grounding + 合规风险」的组合（grok2api 最典型）。

## 风险观察

以下为本周「热度与社区足迹不匹配」或「合规/身份存疑」的样本，用证据说明，不做定性攻击：

- **Graphify-Labs/graphify**：283→291:1（84K→89K ⭐ / subscribers 306），仓龄仅约 102 天，贡献者高度集中（safishamsi 一人 800 提交、第二名仅 21 次），HN/X/Reddit 零有机热度。设计思路（tree-sitter AST 非向量图谱）本身合理、508 个 open issue 有真实工程反馈，但「最高 70x/49x 省 token」的数字来自营销博客（MindStudio/CLSkills）未经独立复现，传播路径明显强于技术验证。
- **ibelick/ui-skills**：358:1（4,292 ⭐ / 仅 12 subscribers），单人主导（ibelick 156 提交、其余各 1-2 次），HN 仅 1 条 story（5 分 0 评论）、Reddit 无讨论，issue 区几乎无独立用户使用反馈（多为第三方想进 registry 的投稿）。热度由作者个人声誉（motion-primitives/prompt-kit 作者）+ skills 生态热点驱动，缺独立采用证据。
- **AIEraDev/Clypra**：265:1（2,648 ⭐ / 仅 10 watchers），仓龄不足 3 个月、2 人团队（作者占 942 提交），核心拖拽功能 #128 报坏、macOS Universal DMG 在 Intel Mac 跑不起（#81），传播主要靠 X 西语 KOL 与 YouTube 安利，HN/Reddit 零讨论。是「传播热度强、社区 grounding 弱」的典型。

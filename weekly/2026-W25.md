## 本周趋势纵览

本周是 W24「去 agent 化」叙事的**证伪周 + 新主题注入周**。W24 末把「去 agent 化是单周波动还是结构拐点」列为头号悬念，本周用一个干净的反转给出答案：前半周去 agent 化走到极致（6/17 是近一周最彻底的一天，agent 本体挂零），但 6/18 agent/skills 框架强势回潮，仅一天就把三天的趋势打回——**这说明 6/8-6/17 的去 agent 化是榜单算法波动叠加成熟项目回流的表象，而非 agent 热度的结构性退潮**。同时本周注入了 W24 完全没有的新增量主线——**开源模型真发版**（GLM-5/LTX-2），叠加 **iroh 1.0 跨三天事件驱动**、**成熟开源 SaaS 替代品两轮回潮**、**创作/视频工具周末成线**，构成「去 agent 化证伪 × 模型层真信号注入 × 基础设施持续回潮」的多线叠加周。

## 重点项目导读

最值得点开的项目（先价值、后 caveat）：

- **zai-org/GLM-5**（旗舰开源 LLM，已迭代 GLM-5.2）— Z.ai 的 744B/753B MoE 开放权重模型，定位「从 Vibe Coding 到 Agentic Engineering」，专攻长周期自主编码。SWE-bench Pro 62.1 超 GPT-5.5、成本约 1/6、MIT 商用、昇腾全栈训练，HN 768/883 分 + Artificial Analysis 开源权重榜首。本周 grounding 最厚的真信号。**值得关注**：API 限流/宕机有多个 issue 实证（#83 连续 2 天近瘫痪），实测分数与跑分的一致性仍有争议。
- **n0-computer/iroh**（Rust，P2P 网络栈）— 用公钥而非 IP「拨号」对方设备，自动 NAT 打洞 + relay 回退，覆盖 ESP32 到服务器全场景。1.0 发版 HN 1,351 分头条 + Holochain/Fedimint/Start9 真采用，本周跨三天信号-噪声比最高的基础设施项目。**值得关注**：1.0 刚发，relay 路径有内存泄漏（#3565）与 RTT 测量 bug（#4201）。
- **Lightricks/LTX-2**（Python，开源音视频生成模型）— 19B（14B 视频 + 5B 音频）一次推理同步出视频 + 音频，免后期配音；原生 4K/50FPS、单卡 4090 可跑。Artificial Analysis image-to-video top-3。**值得关注**：营销页称 Apache 2.0，GitHub 实际是「LTX-2 Community License」（ARR>$10M 需购商业授权），以 GitHub LICENSE 为准。
- **tursodatabase/turso**（Rust，嵌入式数据库）— SQLite 的 Rust 全量重写，保持兼容并补上原生向量搜索、并发写、嵌入式副本。HN 192 分多帖 + 3000+ commits + 100+ 贡献者，database-per-tenant 与 AI/RAG 是真实场景。**值得关注**：官方明确标 BETA 不建议存生产数据，HN 提及曾有客户数据丢失事件。
- **alexzhang13/rlm**（Python，递归语言模型推理库）— MIT CSAIL（含 Omar Khattab）提出，把超长 prompt 当 REPL 变量让模型写代码递归消化，对抗「上下文腐烂」。GPT-5-mini 跑 RLM 在 OOLONG 上正确数 >2× GPT-5 且更便宜，arXiv 论文 + HN 多帖 + Prime Intellect 机构采用。**值得关注**：已有跟进论文质疑「递归本身可能不是主要增益来源」。
- **tw93/Pake + pppscn/SmsForwarder**（工具型基础设施）— Pake 一行命令把网页打包成 5MB 桌面 app（Tauri 原生 WebView，10.7K forks 即 1:5 高 fork 比）；SmsForwarder 把旧机当短信网关转发验证码到钉钉/飞书/Bark。两者比值都 200:1+ 但属成熟工具结构性高比，中文社区实测扎实。**值得关注**：SmsForwarder 转发验证码需关闭系统验证码保护，存隐私/灰产滥用面。

## 如何理解热度信号

本周方法论延续 W24 并细化为可直接套用的三档分层（详见主线 5）：

- **stars 是注意力，不是采用**；trending 排名看的是加权/相对增量而非绝对值——成熟老项目（pytest +14、teslamate +33、hello-algo +71、cypress +13）今日增量个位/十位数却能在榜，是 GitHub trending 时间窗算法的波动性回流，不代表社区出现新关注事件。判断老仓是否真热，看绝对增量 + 外部讨论，而非「是否在榜」。
- **高 stars/subscribers 比值是入口信号，不是 vanity 结论**。本周给出密集的「高比值但真足迹立得住」样本：turso（242:1，HN + 100+ 贡献者）、Pake（205:1，10.7K forks 真打包）、SmsForwarder（211:1，中文社区实测）、GLM-5（141:1，旗舰模型 repo 结构性偏高）、iroh（143:1，早期基础设施库常见）。超大成熟仓（freeCodeCamp 44 万星、puppeteer、insomnia）的高比值同样是结构性正常。
- **真正的 vanity 判据是组合**：短龄 + 单一作者/组织 + 名人/公司/KOL 投放 + 缺第三方独立验证 + 足迹缺失。本周命中的是 palmier-pro（209:1，YC/X 营销 + 单人主导 + 2 月龄）、agent-native（subscribers 仅 2，公司自推 + HN/Reddit 零讨论）、codebase-memory-mcp（187:1 + 营销数字不一致 + 单人主导）、OpenMontage（105:1 + X 纯模板营销 + 6 周停更）、Agent-Reach（366:1，自 6/9 起 vanity 信号持续未收敛）。

### 1. 「去 agent 化」是波动不是拐点（W24 头号悬念收口）

**(W24 悬念收口)** W24 末问「去 agent 化是单周波动还是结构拐点」——W25 用 6/17（agent 本体挂零，去 agent 化极致）→ 6/18（agent/skills 框架一天强势反转）的干净反转给出答案：是波动。三天趋势被一天打回，说明 6/8-6/17 的去 agent 化是成熟项目集体回流挤占头部席位的表象，而非 agent 供给侧枯竭。

### 2. 开源模型真发版重新成为榜单真信号来源（本周新增主线）

**(本周新增)** W24 全周头部无模型层发版，W25 在 6/19 出现 GLM-5/GLM-5.2 + LTX-2 双发版，加上 timesfm 持续在榜。GLM-5 是本周 grounding 最厚的项目（HN 768/883 分 + Artificial Analysis 开源权重榜首 + SWE-bench 超 GPT-5.5 + MIT）。

**信号**：当 agent 工具层多为短龄高比值营销仓时，模型层（GLM-5/LTX-2）和事件驱动基础设施（iroh）是「热度有真实技术验证」的真信号。**判断**：未来观察 (a) 开源模型发版回流是否延续成稳定主线（更多国产/厂商开源模型借发版上榜）；(b) 模型层与 agent 工具层的注意力是否呈此消彼长。

### 3. 真采用 vs 传播型热度的三档分层模板化（W24 预判兑现 + 升级）

**(W24 预判兑现)** W24 把判据收口为「比值是入口信号、结论靠足迹」，W25 在 6/20 把它落成可直接套用的三档分层：真信号（turso/timesfm/GLM-5/iroh/rlm，足迹在 HN/贡献者/机构采用/评测）/ 需结合足迹（palmier-pro/codebase-memory-mcp/OpenMontage/headroom，营销话术 + 弱社区足迹）/ 收藏型（awesome-list，star 反映 bookmark）。

### 4. 事件驱动基础设施延续「双强」形态（W24 apple/container 模式复刻）

**(W24 预判兑现)** W24 的 apple/container（WWDC26 + 1.0 + HN 1,197 分）是「发布会触发 + 社区自发验证」双强样本，W25 的 iroh（1.0 + HN 1,351 分 + Holochain/Fedimint 真采用）跨三天复刻这一形态。

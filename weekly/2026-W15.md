## 值得关注的信号

### 1. Claude Code 生态已形成"工具全家桶"效应

一周内 9+ 个 Claude Code 生态项目上榜不是巧合——当一个 CC 项目上热榜，会连带拉动"CC 工具全家桶"整体曝光。andrej-karpathy-skills 的病毒式传播（4 天内增量 +51→+2,369 指数增长）带动了 best-practice、claude-mem、seomachine 的回潮。这个正反馈循环是 Claude Code 生态区别于 Cursor/Copilot 的独特现象。

### 2. NousResearch/hermes-agent 可能是下一个 "superpowers 级别" 的项目

从 03-30 首次上榜到现在连续 6+ 次上榜，增量不减反增（+1,574 → +6,485 → +7,454），这种增长曲线与 obra/superpowers 早期（03-30 上榜后增长到 143K）高度相似。hermes-agent 目前的绝对 star 数远低于 superpowers，但增量已经超过它——后续走势值得关注。

### 3. 文档预处理正在成为独立赛道

opendataloader-pdf（benchmark 0.90，Java）和 VoxCPM（Tokenizer-Free TTS）的同时上榜，加上 W16 开头的 markitdown（104K stars），说明 RAG 管道中"把各种格式转为 AI 可用数据"这个环节已经从"附属功能"升格为"独立赛道"。竞争格局：markitdown（格式覆盖广）vs opendataloader-pdf（PDF 精度高）vs Docling/Marker/LlamaParse。

### 4. "学术项目热启动"模式频繁出现

本周 Kronos（AAAI 2026）、DeepTutor（HKU）、VoxCPM（清华）、newton（Disney/DeepMind/NVIDIA）四个学术/大厂联合项目同时上榜。共同特征：论文驱动 + 中文社区传播 + HN 低讨论 + 停更风险。提示：关注论文发布后 3 个月内的维护活跃度，这是区分"论文项目"和"真正开源项目"的关键窗口。

### 5. Stars 泡沫指标越来越极端

本周多个项目的 Stars/Watchers 比严重失衡：GitNexus 23.5K stars 但 HN 零评论、shannon 36.6K stars 但 HN 4 分、Kronos 12K stars 但代码 3 月未更新。社交媒体（X/YouTube/GitHubDaily）驱动的 star 膨胀已成为常态——watchers 数和 HN 讨论量是更可靠的"真实关注度"指标。
